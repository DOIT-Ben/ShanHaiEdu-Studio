# v0.2 工作台视觉与前端架构重构规划

## 1. 第一性原理

当前目标不是继续堆卡片，而是让教师在一个低密度、浅白、主会话优先的工作台里完成线性公开课生产。界面必须满足：

- 左侧只负责项目管理。
- 中间是主要对话和当前任务执行区。
- 右侧节点默认收起，只显示线性进度与当前位置。
- hover 展示缩略信息，点击才打开详情抽屉。
- 生成、失败、重审、详情等状态要成体系，不把状态逻辑散落到一个大组件里。

## 2. 复用与参考

本轮参考用户提供的目标图与 `shanhai-state-splits.zip` 中 8 张状态图：

- `02-ppt-pages-generating.png`：资源生成态，中央大生成卡 + 右侧节点高亮。
- `08-artifact-detail-drawer.png`：节点点击后右侧详情抽屉，包含基本信息、缩略图、可复制内容。
- `07-failure-recovery.png`：失败恢复态，中央明确说明损失情况和下一步动作。

工程复用现有 Next/Tailwind/Radix/lucide，不引入新 UI 框架。保留现有 shadcn 风格基础组件，只补业务组件拆分。

## 3. 复用、适配和必要自研

复用：

- `Button`、`Badge`、`Sheet`、`Popover`、`ScrollArea` 等现有基础组件。
- `mock-data.ts` 的项目、产物和消息数据。

适配：

- 全局色板改为浅白灰 + 深青绿 + 柔和橙 + 低饱和红。
- 中央工作区采用目标图的宽画布、阶段条、消息流和生成卡。
- 右侧节点轨道从“卡片栏”改成“带标签的垂直节点脊柱”。

必要自研：

- `WorkbenchTopbar`：面包屑、保存状态、协作入口。
- `StageProgress`：5 步阶段条。
- `ChatTranscript`：对话消息区。
- `GenerationPanel`：当前生成任务卡，支持 PPT/图片/视频/失败状态。
- `ArtifactSpine`：右侧收起节点串和 hover 预览。

## 4. 开发方案、风险和验证

开发方案：

1. 拆分 `ConversationWorkbench.tsx`，减少单文件职责。
2. 拆分右侧节点轨道，让 `ArtifactNodeCard` 只负责单节点触发与预览。
3. 更新全局 token 和左侧项目栏，贴近浅白低密度目标图。
4. 增加中央生成卡和右侧浮动节点预览。
5. 保留现有 copy/use-as-input/detail/confirm/retry 交互。

风险：

- 视觉参考图中有大量阴影和浮层，过度复制会使页面显得花；本项目只保留必要层次。
- 图片缩略图没有真实资产，当前用可编辑 CSS 预览块模拟，必须继续标注演示态。
- 拆分文件后要避免“组件过碎”，只拆有明确职责和复用价值的业务块。

验证标准：

- `npm run build` 通过。
- 桌面 1440px：左项目栏、中间会话主画布、右节点脊柱三块清晰。
- 右侧节点 hover 展示缩略信息，点击打开详情。
- 390px：项目和产物可通过抽屉访问，主会话不横向溢出。
- 单个新增业务组件保持在约 150 行以内。
