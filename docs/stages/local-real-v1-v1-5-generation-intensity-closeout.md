# ShanHaiEdu V1-5 四档生成强度收尾

更新时间：2026-07-13

状态：`done`

## 1. 完成内容

- Project 持久化强度与乐观版本；新项目默认标准档。
- ConversationTurnJob 入队时冻结强度与版本，项目后续改档不影响旧任务。
- Main Agent 与专业 Agent Tool 都从同一任务快照解析内部模型策略。
- 教师端只显示标准、增强、深度、极致和相对积分趋势，不显示模型、推理参数或生成服务名称。
- 标准、增强、深度直接受控写入；极致先返回绑定项目、版本和目标档的确认 action，确认后写入，旧 action 因版本变化不能重放。
- 服务端根据持久化任务的连续失败状态只建议下一档；深度档不足持续失败门槛时不建议极致。建议不会自动写入，仍由教师确认。
- Composer 旧“选择模型”占位已替换为真实生成强度入口；设置弹窗提供四停靠点 slider、键盘操作、相对消耗提示和极致独立确认。

## 2. 修正的问题

浏览器验收发现快速连续调档时，第二次请求可能携带旧版本。服务端正确返回 409，但前端曾产生未处理异常。现已由 controller 持有最近一次服务端确认版本，冲突时显示教师可读提示并恢复当前服务端档位。
