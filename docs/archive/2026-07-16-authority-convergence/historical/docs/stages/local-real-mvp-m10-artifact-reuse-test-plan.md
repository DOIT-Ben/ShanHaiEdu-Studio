# Local Real MVP M10 产物复用输入闭环测试定义

日期：2026-07-07

## 1. 测试目标

M10 测试必须证明教师能从产物详情页把材料复用到下一轮对话，并且复用引用进入真实后端消息，而不是只停留在前端视觉状态。

## 2. T1：详情页作为输入回到 composer

前置：

- 新建项目。
- 输入一句话公开课需求。
- 等待需求规格说明书生成。
- 打开“需求规格说明书”完整详情页。

步骤：

1. 点击详情页底部“作为输入”。
2. 检查完整详情页关闭。
3. 检查 composer 仍可见。
4. 检查 composer 显示“引用：需求规格说明书：已整理公开课目标、基础信息、交付范围和后续输入要求。”。
5. 检查 textarea value 包含“请基于：需求规格说明书｜已整理公开课目标、基础信息、交付范围和后续输入要求。”。

通过标准：

- 输入区没有被详情抽屉遮挡。
- 引用 chip 与 textarea 内容都存在。
- 教师界面不出现工程词。

## 3. T2：发送后保存引用并回显

前置：

- T1 已完成，composer 已带引用。

步骤：

1. 保留引用 chip，将输入正文替换为“请继续细化课堂活动。”。
2. 点击“发送”。
3. 等待 `/api/workbench/projects/:projectId/messages` 返回 201。
4. 请求 `/api/workbench/projects/:projectId/messages`，检查最后一条 teacher message 的 `artifactRefs` 包含该引用。
5. 检查页面教师消息同时显示正文和“引用：需求规格说明书：...”。

通过标准：

- 后端返回成功后页面出现带引用的教师消息。
- 引用来自 `artifactRefs` 保存后的 snapshot，不依赖本地临时状态。

## 4. T3：移除引用不清空输入正文

前置：

- 已有可复用产物。

步骤：

1. 再次打开需求规格说明书详情页。
2. 点击“作为输入”。
3. 在 textarea 中追加“下一步只关注学生活动。”。
4. 点击“移除引用”。
5. 检查“移除引用”按钮消失。
6. 检查 textarea 仍包含“下一步只关注学生活动。”。

通过标准：

- 移除引用只影响引用 chip。
- 已编辑输入不丢失。

## 5. 回归范围

集中验收必须覆盖：

- Stage 2 Chromium desktop 主链路。
- Stage 8 Chromium narrow 与 Firefox desktop 覆盖。
- Stage 7 双 browser context 隔离。
- Node/Vitest 总测试。
- Build。
- Markdown 下载专项测试。

## 6. 不测范围

- 不验证真实 OpenAI live smoke，除非本机已配置真实凭据。
- 不新增 WebKit 或真实移动设备专项。
- 不测试真实 PPTX、图片或视频文件生成。
