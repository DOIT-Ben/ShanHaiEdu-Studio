# 04 视频生产工艺与质量架构

> 2026-07-13 V1口径说明：本文是迁入的历史设计底稿。当前正式顺序是先让独立创意成立，再提取唯一最小课程锚点；课程锚点不得把“小学生受众”扩张成儿童主角、教室或课堂活动的必需条件。课程锚点必须由独立`delivery_critic.review`审查，真实媒体整包验证延后到V1-9。冲突时以`docs\product\current-requirements-baseline.md`和V1主线调整计划为准。

## 1. 什么才算“好导入视频”

ShanHaiEdu 的导入视频不是缩短版微课。它必须：

- 有独立创意和第一时间钩子，但通过课程锚点自然回到课堂。
- 只制造观察、冲突、悬念或问题，不提前把知识答案讲完。
- 每个镜头有明确功能、主体、动作、运动、首帧/终帧和连续性。
- 参考资产真实参与 Provider 请求，而不是只存在于数据库。
- 数字、公式、答案、复杂中文优先在后期可控层完成。
- 每个镜头独立生成、审查、返修和追踪，不因一镜失败重做整片。
- 成片音画、字幕、时长、编码和播放结构真实可验收。

## 2. 推荐双路径

### Short Preview

适合 6-15 秒创意样片或课堂单镜头素材：

```text
课程锚点 → 单镜头 CreativeBrief/ShotSpec → 参考资产（按需要）
→ 单镜头生成 → 技术 QA + 内容 QA → 教师预览
```

必须记录 `productionPath=video_short_preview`、`videoMode=short_preview`、`deliveryEligibility=preview_only`，并标记为“单镜头预览”，不能冒充完整导入视频或进入最终材料包。

### Full Intro

```text
课程锚点
→ 独立创意选择
→ 文稿与节拍表
→ 导演分镜 ShotSpec[]
→ ReferenceAssetManifest
→ 首帧/终帧门禁
→ ProviderProfile 解析
→ 按 shotId 逐镜头生成
→ 单镜头 ffprobe + Critic + 变量化返修
→ 中文旁白 / 叠字 / 字幕
→ 编码归一化
→ FFmpeg 时间线合成
→ 成片 Validator + Critic
→ 教师批准
```

Full Intro 必须记录 `productionPath=video_full_intro`、`videoMode=full_intro`。完成时间线、ValidationReport、CriticReport 和教师决定后，才可成为 `final_eligible`。Short Preview 的已批准创意、ShotSpec 或专用视频资产可以在血缘合法时复用，但产物本身不能靠改字段“升级”为 Full Intro。

## 3. 逐节点工件、门禁和回退

| 阶段 | 必须产出 | 硬门禁 | 失败回退 |
|---|---|---|---|
| 课程锚点 | `VideoIntent` | 来自已批准教案/教材；含易错点、课堂回接问题、答案泄露边界 | 重新提取或询问教师 |
| 独立创意 | `CreativeBrief` | 与锚点有关、不提前授课、适龄、安全、有可视化机制 | 换创意，不用加长 prompt 硬顶 |
| 文稿节拍 | `ScriptBeatSheet` | 视觉节拍、旁白、叠字、字幕、目标时长区间分层；复杂精确信息后期化 | 压缩/重排节拍 |
| 导演分镜 | `StoryboardManifest + ShotSpec[]` | `shotId` 唯一；每镜一个主要叙事功能；开始/结束状态可描述；连续性规则明确 | 过载镜头拆分，空洞镜头合并 |
| 参考资产 | `ReferenceAssetManifest` | `assetDomain=video`；assetId、用途、hash、尺寸、来源、适用 shot；禁止 PPT 资产/PPT contact sheet/候选视频抽帧血缘 | 只重做缺失资产；跨域资产直接阻断 |
| 首尾帧 | `FrameGateReport` | 连续性/身份关键镜头的参考策略满足；失败镜头不得提交 | 修参考图、构图或 ShotSpec |
| Provider 解析 | `ResolvedShotExecution[]` | 模型能力、时长、比例、参考图传输方式、首尾帧、成本相容 | 换 Profile 或重规划镜头 |
| 逐镜头生成 | `GenerationJob + ShotClip` | 每个 job 绑定 shotId、inputHash、providerTaskId、attempt；真实参考图已传 | 只重试失败 shot |
| 单镜头 QA | `ShotValidationReport + ShotCriticReport + ShotQualityDecision` | ffprobe 硬门通过；Critic 检查主体、动作、终帧、连续性、安全和答案泄露 | 优先修改报告定位的 retakeVariables；关联变量可最小联动并说明理由 |
| 音字后期 | `PostProductionPackage` | 中文旁白、叠字/字幕事实正确；时间轴同步；安全区可读 | 只返修音轨/字幕/overlay |
| 编码归一化 | `NormalizedClipManifest` | 分辨率、FPS、像素格式、编码、音频采样率/channels、time base 一致 | 对问题 clip 转码 |
| 时间线合成 | `TimelineManifest + FinalVideoArtifact` | 按 `shot.ordinal` 合成；镜头完整；真实可播放；时长与音轨正确 | 返回缺失/异常 shot 或 post track |
| 成片审查 | `VideoValidationReport + VideoCriticReport + VideoQualityDecision` | Validator 技术门通过；Critic 判断教学、叙事、连续性、音字和安全；Decision Engine 聚合 | 定位 shotId/track/timeline/responsibleStage |
| 教师批准 | `HumanDecision` | 绑定 productionPath、artifact version、inputHash 和 action；硬门/blocker 不可豁免 | 保留上一选定版本，按反馈局部返修 |

## 4. ShotSpec 最小语义

```text
shotId
ordinal
durationTargetRange
sceneFunction
mainSubject
shotSize
cameraAngle
compositionFocus
continuityKeys
startFrameIntent
endFrameIntent
endFrameHoldMs
subjectAction
cameraMotion
lightingAndStyle
referencePolicy
referenceAssetIds
textPolicy
voiceoverSegmentId
modelPrompt
negativePrompt
retakeVariables
```

镜头数和时长不是全局固定值。`6 × 10 秒` 是一分钟预设，`omni_flash-10s` 是旧 ProviderProfile 默认，不应进入不可覆盖合同。

`referencePolicy` 建议取：

- `required`：身份、角色、道具、场景连续性或首尾帧关键镜头。
- `recommended`：参考图会显著提升一致性，但模型支持不足时可解释降级。
- `none`：纯转场、抽象环境或参考图可能抑制创意的镜头。

这比“所有镜头一律必须有参考图”更能保留 Agent 的导演判断，同时对真正需要连续性的镜头保持硬约束。

## 5. Video Skill 应包含什么

只建立一个可路由 Skill：`edu-intro-video-production`。

建议章节：

1. 课程锚点与不替代授课
2. 独立创意模式和适用场景
3. 节拍表、旁白/叠字/字幕分层
4. 镜头拆分、导演参数和运动词汇
5. 参考资产、首尾帧和连续性策略
6. ProviderProfile 适配和降级策略
7. 单镜头 QA 与 retakeVariables
8. 音字后期、编码和时间线合成
9. 成片审查和返修路由

模型名、网关地址、参考图数量上限、分辨率和并发不写在 Skill 通用方法中，而由 ProviderProfile 提供。

## 6. 质量量表

建议成片总分 100：

| 维度 | 权重 | 评价重点 |
|---|---:|---|
| 教学锚点与不泄题 | 25 | 课程关联、课堂回接、不替代授课、事实正确 |
| 钩子与叙事 | 15 | 第一时间吸引、冲突/悬念、节拍递进、结尾设问 |
| 镜头导演与画面可读 | 15 | 一镜一功能、主体动作、镜头运动、构图和视觉焦点 |
| 主体/风格连续性 | 15 | 角色、道具、场景、方向、光线和终帧衔接 |
| 旁白/叠字/字幕 | 10 | 中文正确、同步、可读、精确信息不由视频模型生成 |
| 安全与课堂适配 | 10 | 儿童安全、隐私、品牌/版权、无不当视觉和音频 |
| 技术交付 | 10 | 可播放、镜头完整、时长、分辨率、音轨、编码、容器 |

建议：ValidationReport 必须全过；单镜头 Critic 通过线 >= 80；成片 >= 85；无 blocker；任一确定性硬失败直接 block。

硬失败包括：答案泄露、儿童隐私/不当内容、必需参考资产未真实传递、缺镜头、文件不可播放、错误语言音轨、字幕/叠字事实错误、时间线顺序错误。

## 7. 当前视频 releasePriority=P0 缺口

### 7.1 参考图没有传给 Provider

`EVOLINK_GROK_IMAGINE_VIDEO_PROVIDER_PROFILE` 声明最多 7 张 `image_urls`，但 `buildVideoRequestBody` 只发送 model、prompt、duration、quality、mode、aspect ratio；`asset_image_generate` 只用于检查前置工件存在。

优化：Tool 输入必须是 `ResolvedShotExecution`，Provider Adapter 负责按 Profile 使用 multipart 字节或受信 URL 真实传参考图，并把已传 assetId/hash 写进 job provenance。

### 7.2 不是逐镜头调用

当前一次生成读取整份 `video_segment_plan` Markdown 前 1600 字，没有 `shotId`、ordinal、单镜头 prompt 和单镜头 attempt。

优化：每个 shot 建独立 job/result/QA；批次只负责调度，不能把整份 storyboard 当一个视频请求。

### 7.3 数据模型无法同时批准多个镜头

`approveArtifact` 会取消同一 node 旧版本的批准，而 concat 只读取已批准 `video_segment_generate`。这使真实仓储无法同时保留多个可拼接镜头；当前单元测试中手工构造多个 approved 片段绕开了仓储不变量。

优化：镜头不是同一 node 的互斥版本。需要一等 `unitId/shotId`，批准唯一性改为 `(projectId, nodeKey, unitId)`；每个 shot 可有多个版本，但只能选择一个 approved clip。

### 7.4 `Buffer.concat` 不是 MP4 合成

独立 MP4 有各自 `moov/mdat` 索引，字节相接不能形成可靠时间线。

优化：

1. 先用 ffprobe 获取 streams、duration、codec、fps、time base。
2. 对不一致 clip 做 FFmpeg 归一化。
3. 无需转码且参数一致时用 concat demuxer；否则用 concat filter/明确转码。
4. 合成后再次 ffprobe，并验证总时长、镜头数、音视频流和可解码性。

### 7.5 资产工具只生成一张泛化低质量图片

当前图像工具一次生成一张 `1024x1024`、`quality: low` 图片，prompt 仍是“PPT 导入页主视觉”。

优化：参考资产必须按 assetId 批次生成，区分角色、道具、场景、首帧、终帧和文字安全区；每项单独验证、选择和重试。合同必须保存 `assetDomain=video`、source lineage 和 usage policy，禁止 PPT 资产、PPT contact sheet 或候选视频抽帧作为视频参考图。

### 7.6 外部任务无法恢复

Provider taskId 只在同步轮询函数局部存在，GenerationJob 没有保存 taskId、unitId 和 request hash。进程中断后只能重新提交，可能重复计费。

优化：submit 与 poll 解耦；job 保存 providerTaskId、poll state、expiresAt、inputHash；恢复时先查询旧任务。

### 7.7 质量门过浅

当前只检查 MP4 大小和 `ftyp/moov`；没有 ffprobe、镜头内容、连续性、音字和儿童安全审查。

优化：文件真实性只是 Validator 第一层；再由独立 Critic 看真实帧采样/contact sheet、字幕和音频转写证据。

## 8. 当前代码差距表

| 当前事实 | 优化后的状态 |
|---|---|
| 视频节点合同为 0 | 每个关键阶段有 Contract；shot 级合同单独版本化 |
| ProviderProfile 常量未驱动决策 | Runtime Capability Resolver 真实选择传输方式、模型、时长和成本 |
| 参考图只做存在性门禁 | 参考图字节/URL 真实进入请求，并记录 hash |
| 一次计划一次视频请求 | 一个 shot 一个幂等 GenerationJob |
| 一个 node 只能批准一个 Artifact | 每个 shot 独立 selected/approved clip |
| Artifact version 作为镜头顺序 | `shot.ordinal` 是唯一时间线顺序 |
| Buffer.concat | ffprobe → normalize → FFmpeg concat → post-validate |
| 无后期音字 | TTS、字幕、overlay 作为可单独返修的轨道 |
| 无独立 Critic | ShotCritic + FinalVideoCritic，均只读真实证据 |

## 9. 视频侧 releasePriority=P0 验收标准

1. 三个 shot 能分别提交、恢复、失败重试和批准，不互相覆盖。
2. 至少一个 identity/continuity 镜头能证明参考图真实进入 Provider 请求。
3. PPT 资产、PPT contact sheet 和候选视频抽帧作为 reference input 时，ContractValidator 能稳定阻断。
4. 故意中断轮询后可凭 providerTaskId 恢复，不重复提交。
5. 不同 codec/fps 的片段能被识别、归一化和正确合成。
6. 合成视频的 ffprobe 总时长、音视频流和镜头清单与 TimelineManifest 一致。
7. Critic 能把连续性、答案泄露或字幕问题定位到 shotId/track，但不能重判 ffprobe 等确定性结果。
8. Short Preview 在 FinalDeliveryGate 被机器判定为 `preview_only`。
