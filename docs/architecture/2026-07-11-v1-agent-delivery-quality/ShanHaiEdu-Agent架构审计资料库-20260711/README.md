## 核心结论

当前证据支持以下分层判断：

1. 不建议继续自研通用 Agent Runtime，包括通用 ReAct 循环、条件路由、Checkpoint、Interrupt/Resume、子图调度和通用 Tracing。
2. 不应删除 ShanHaiEdu 自研业务 Main Agent。教师意图理解、PPT/视频生产工艺、教材证据、Artifact 版本、质量判断和最终交付资格仍是自研核心。
3. 当前首选候选为 LangGraph OSS Core；OpenAI Agents SDK 是 Agent-as-Tool 场景的第二候选；Vercel AI SDK 更适合 Next.js 流式交互层。
4. 不建议第一阶段同时引入三套框架，也不建议一次性重写现有 ToolRouter、Provider Adapter、Artifact、Job 和质量体系。
5. 第一性原理复核发现，现有方案还缺少“教师委托与信任层”：版本化委托合同、变更影响预览、真实课堂演练、输入侧数据/版权治理和教师可见的自主度/成本/时间协议。
6. 快速上线 V1 建议保留当前宏观架构，先完成 `v1` / M72-M78 标签验证封板，再补最小运行护栏并闭环 PPT Quality、视频 Full Intro 和最终交付；框架迁移后置。
7. 2026-07-11 23:01 后基线已变化：M72-M78 已提交并打 `v1` 标签，旧“脏工作树交接”改为“标签验证封板”；在 closeout、浏览器、真实 Provider、服务器和教师门关闭前，标签只代表工程候选。
