# 目标模式 Hand off 模板

```md
你接手 `【项目名】` 的 `【主线名】` 主线，进入目标模式持续推进，直到本主线完成、验证通过、提交并给出可合并结论；不要只写计划后停止。

工作目录：`【绝对路径】`
本地分支：`【feature/xxx】`
远程分支：`origin/【feature/xxx】`

先执行并确认：
`git status --short --branch`
`git branch -vv`

先读：
1. `AGENTS.md`
2. `【需求文档】`
3. `【总体架构文档】`
4. `docs\mainlines\README.md`
5. `【本主线规划文档】`

可用私有资料：
- 本地 API 台账见 `docs\private-api-ledger.md`。需要接入 API 或确认 provider 能力时可查，但不得提交、摘录或泄露其中的密钥和敏感配置。

主线目标：
- 【目标 1】
- 【目标 2】
- 【目标 3】
- 【目标 4】

执行规则：
- 先把整条主线拆成阶段，原则上不超过 20 个阶段。
- 每个阶段都按：阶段规划文档 -> 阶段测试文档 -> 集中开发 -> 集中测试 -> 修复复测 -> 阶段收尾 -> 提交。
- 阶段内可以做必要快速检查，但阶段验收必须等该阶段开发完成后集中执行。
- 一个阶段完成并提交后，再进入下一阶段。
- 不堆补丁；必要时先重构。
- 不污染 `main`。
- 不把 mock / placeholder / deterministic 输出伪装成真实完成。

推荐第一阶段：
【写清第一阶段要做什么，要求产出哪些 plan/test 文档，然后继续开发、验收、提交】

边界：
【写清本主线不做什么；跨主线问题只记录接口需求或阻塞，不越界重写】

停止条件：
- 本主线目标全部完成、测试通过、已提交，并说明是否可合并到 `main`。
- 或同一外部阻塞连续三轮无法绕过，并写清事实、阻塞、尝试、下一步。
```
