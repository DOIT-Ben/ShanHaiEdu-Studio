# Local Real MVP M12 最终交付清单识别 PPTX 下载能力测试定义

日期：2026-07-07

## 1. 测试目标

M12 测试必须证明最终交付清单已经同步 M11 的真实能力边界：PPT 大纲可以下载最小 PPTX 文件，但图片、视频、动画和视觉精修仍未完成。

## 2. T1：后端最终交付 artifact 文案

步骤：

1. 创建项目。
2. 发送一句话公开课需求。
3. 依次确认需求规格、教材证据、教案、PPT 大纲、导入视频方案。
4. 读取 `final_delivery` artifact。
5. 检查 Markdown 包含：
   - “PPT 大纲可下载最小 PPTX 文件”
   - “图片文件、视频成片、动画和视觉精修仍待生成或完善”
6. 检查 Markdown 不包含：
   - “PPTX 文件已生成”
   - “图片文件已生成”
   - “视频成片已生成”

通过标准：

- 最终交付清单能准确描述 M11 的最小 PPTX 能力。
- 仍不伪装图片、视频或精修能力。

## 3. T2：Markdown 下载内容同步

步骤：

1. 使用最终交付清单样例构建 Markdown 下载。
2. 检查下载内容包含 PPTX 最小下载能力提示。
3. 检查下载内容不包含虚假完成表述。

通过标准：

- 浏览器下载的 `.md` 交付清单与详情页口径一致。

## 4. T3：浏览器详情页与下载验收

步骤：

1. 运行 Stage 2 主链路到最终交付清单。
2. 打开最终交付详情页。
3. 检查详情页可见 PPTX 最小下载能力提示。
4. 下载 Markdown。
5. 检查下载内容包含 PPTX 最小下载能力提示。
6. 检查页面和下载内容仍不出现虚假完成表述。

通过标准：

- 教师在页面和下载文件中看到一致的真实能力边界。

## 5. 回归范围

- 后端 workflow final delivery 测试。
- Markdown 下载专项测试。
- Stage 2 Chromium desktop。
- 总测试 `npm test`。
- 生产构建。
- Stage 8 窄屏与 Firefox 覆盖。
- Stage 7 双上下文隔离。
