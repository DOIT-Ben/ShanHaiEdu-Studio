# 落地优先级与产品路线

## 1. 总体判断

不需要重写 Agent OS，也不需要先换 Provider。现有 ToolRouter、Artifact Truth、HumanGate 和状态持久化应继续作为底座；但必须先把“一轮一个工具计划”升级为受控 ReAct 循环，否则 Contract 和评价闭环仍会被固定链路限制。

## 2. P0：两周内最高收益

### P0-0 补齐受控 ReAct 主循环

- 引入 `AgentLoopPort`，让 Tool Result 在同一 Run 内回到 Main Agent。
- Plan 只作为可选、可修改的 Working Plan，不作为固定顶层 DAG。
- 非致命失败必须成为 Observation，允许 Agent 换输入、换工具、追问或局部返工。
- 增加 Checkpoint、Interrupt、AgentEvent、稳定 invocation ID 和幂等键。
- WorkflowNode 只保留教师可见里程碑语义。

### P0-1 增加效果 Contract

- PPT：Deck Narrative、PageSpec、Render Review。
- 视频：Concept Selection、Beat Sheet、ShotSpec、Final Review。
- 交付：Manifest Truth Audit。

### P0-2 增加生成者/审查者分离

- 同一模型可以不同会话承担角色，但审查上下文不得包含生成者自评。
- Critic 必须输出结构化 Finding。
- Director 与 Critic 优先作为 agents-as-tools，由 Main Agent 保留教师会话控制权。

### P0-3 修复视频合成业务规则

- 将“只拼接”定义为正规的媒体时间轴拼接，而不是文件字节相加。
- 增加总时长、流一致性和完整解码门。
- 将转码、时间轴、字幕、音轨和技术校验封装为确定性 Workflow Capsule Tool。

### P0-4 增加真实目录反向审计

- 最终 README/manifest 只能由实际文件和校验记录生成。

## 3. P1：形成稳定效果闭环

### P1-1 逐页主视觉

把 `generate_classroom_image` 从“一套 PPT 一张图”升级为“PageSpec 驱动的按需视觉任务”，但共享背景可以复用。

### P1-2 视频 Asset Bible

资产说明不再只产一张图，而是产风格、角色、场景、道具和关键帧引用集合。

### P1-3 渲染后自动审查

- PPTX -> PDF/PNG -> deterministic + multimodal review。
- MP4 -> 抽帧/联系表/音频统计 -> multimodal review。

### P1-4 局部返工

建立 page_id / shot_id 级 lineage，避免整套重生成造成已通过内容漂移。

## 4. P2：Provider 与评估优化

- 用固定教材任务集比较不同 LLM、图片和视频 Provider。
- 评价维度固定后，再做模型、temperature、reasoning、分辨率、片段长度和参考图策略实验。
- 建立黄金样例与回归集，而不是靠一次 demo 判断 Provider。
- 在统一 `AgentLoopPort` 后对比 ResponsesLoopRuntime 与 AgentsSdkRuntime；不同时叠加 LangGraph。

## 4.1 框架接入顺序

1. 先扩展现有 Responses API 自管循环，保留当前 OpenAI-compatible Provider 和 ToolRouter。
2. 再用相同契约隔离试点 OpenAI Agents SDK Runner；通过兼容性、恢复、成本和真相层验收后才可切默认。
3. LangGraph JS 只在 Checkpoint/Interrupt 自研复杂度明显上升时做纵向试点，不建立第二套业务状态。
4. AutoGen、Google ADK、Claude Agent SDK 当前不引入，只抽取其 handoff、event loop、hook 和 tool design 思想。
5. Temporal 等到跨进程、多小时 Worker 恢复成为真实需求后再接入 `DurableJobCoordinator`。

## 5. 推荐指标

### 产品指标

- 教师首次可用率：无需大改即可授课的交付占比。
- 页级返工率 / 镜头级返工率。
- 从需求确认到通过质量门的时间和成本。
- 教师修改集中在哪些 Rubric 维度。
- 一次完整交付中整链重跑次数，应趋近于 0。

### PPT 指标

- PageSpec 落实率。
- 文字溢出、低对比和小字号页数。
- 连续重复版式比例。
- 主视觉解释力评分。
- 教材/数学错误数量。

### 视频指标

- ShotSpec 落实率。
- 黑帧、冻结、静音、字幕冲突时长。
- 角色/场景连续性失败镜头数。
- 课程锚点命中率。
- 片段时长总和与成片时长偏差。

## 6. 建议试点

选择 3 个难度递增任务：

1. 低年级数概念：精确数量和简单叙事。
2. 中年级几何或测量：过程动画和多表征。
3. 高年级百分数/分数：教材证据、关系图和生活迁移。

每个任务固定输入、目标页数和视频时长，分别跑现有流程与 A+ 流程，盲评 PPT/视频，不让审查者知道版本。

## 7. 上线门

达到以下条件后才建议进入真实教师内测：

- 3 个试点全部完成真实 PPTX 与 MP4。
- Artifact Truth 和 Delivery Effect Truth 均通过。
- 没有 README/manifest 与目录不一致。
- P0 缺陷为 0，P1 缺陷有明确处置。
- 至少一名真实教师完成课堂演练或结构化可用性评审。
