# v0.2.2 Codex 式三栏统一规划

## 1. 第一性原理

本轮核心是把“每一栏都有自己的视觉语法”收敛为同一套工作台语法。教师不需要被卡片边框、粗字、状态徽标和局部颜色牵引，界面只需要表达三件事：当前位置、当前对话、可复用产物。

可验证标准：

- 字体栈统一为 Windows 上接近 Codex 的 `Segoe UI Variable / Segoe UI`，中文回退到 `Microsoft YaHei UI`。
- 字号只保留 12px、14px、16px 三档。
- 左侧项目栏不再像卡片列表，改为浅灰栏里的紧凑行列表。
- 选中态只使用浅灰底，不使用彩色文字、彩色点或强边框。
- 右侧节点栏与左侧一样使用浅灰栏，不再额外套一个显眼胶囊容器。
- 中央仍是主视觉，输入框和任务卡只保留轻边框与轻阴影。

## 2. 参考与复用

参考用户提供的 Codex 侧栏截图：

- 侧栏背景是统一浅灰，不是纯白卡片堆叠。
- 列表项默认无边框，选中才有浅灰底。
- 字体字重克制，项目名不需要大面积加粗。
- 操作按钮存在感低，靠位置和图标表达。

继续复用现有组件和拆分架构，不引入新 UI 库，不重写业务状态。

## 3. 适配方案

- 全局字体改成 Codex 接近的系统字体栈。
- `ProjectSidebar` 改成行列表：标题、状态、时间在同一浅灰语法里展示。
- `ArtifactRail` 去掉显眼外框，改为浅灰栏上的线性节点。
- `ArtifactNodeCard` 当前项只用浅灰底和正常字重。
- 阶段条、会话标题、生成卡标题整体从 `font-semibold` 降到 `font-medium`。

## 4. 风险与验证

风险：

- 降低字重后层级可能变弱，所以保留 16px 标题与位置结构。
- 右侧去掉外框后节点可能弱化，所以保留竖线和当前节点浅灰底。

验证：

- `npm run build` 通过。
- 桌面 1440px 截图中三栏灰阶一致，没有局部彩色噪点。
- 390px 无横向溢出。
- 右侧节点仍可点击打开详情。
