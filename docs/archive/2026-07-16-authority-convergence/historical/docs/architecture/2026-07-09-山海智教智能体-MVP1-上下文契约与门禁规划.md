# 山海智教智能体 MVP1：上下文、契约与门禁骨架实施规划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立山海智教智能体的第一阶段骨架：完整保留对话历史、后台无感压缩旧上下文、每轮模型只读取 `ContextPackage`、关键节点规则文件化、模型计划经过门禁后才执行。

**Architecture:** 本阶段不重写完整工作台，而是在现有 `conversation-turn-service`、`model-main-conversation-agent`、`capability-registry`、`workbench repository` 周围增加清晰边界。先落 `Conversation Context MVP`、`Node Contract MVP`、`PlanGuard/HumanGate MVP` 三条主线，保留现有真实 PPTX、图片、视频、Artifact 链路。

**Tech Stack:** Next.js 16 App Router、React 19、TypeScript、Prisma 7 + SQLite、Vitest、OpenAI-compatible Runtime、现有 Workbench Service/Repository。

## 2. MVP1 范围

### 2.1 本阶段做

1. 新增 `ContextPackage` 数据结构和构建器。
2. 新增上下文预算估算和压缩触发策略。
3. 新增 `SessionContextSnapshot` 和 `ContextBuildLog` 持久化模型。
4. 新增后台 `SessionCompactor` 接口和 deterministic/test 实现。
5. 新增 `SummaryValidator`，防止摘要伪造完成状态、丢关键约束、污染长期记忆。
6. 新增文件化 `NodeContractRegistry`，先覆盖关键节点：
   - `requirement_spec`
   - `lesson_plan`
   - `ppt_design`
   - `coze_ppt`
   - `final_package`
7. 新增 `PromptCompiler`，让模型输入优先来自 `ContextPackage + NodeContract`。
8. 新增 `PlanGuard` 和 `HumanGate actionId` 最小实现。
9. 将 `conversation-turn-service` 调整为：先构建上下文包，再调用主 Agent；执行工具前先过门禁。

### 2.2 本阶段不做

1. 不做复杂 Admin 可视化后台。
2. 不做完整组织权限体系。
3. 不重写全部视频生产链路。
4. 不一次性契约化所有节点。
5. 不自动写入长期教师画像。
6. 不删除原始 Conversation Log。
7. 不把 deterministic draft 伪装为真实生成。

```ts
export type ContextBudgetMode = "full" | "precompact_async" | "compact_required";

export type EstimateContextTokensInput = {
  systemRules: string;
  messages: string[];
  artifacts: string[];
  snapshot?: string;
};

export function estimateContextTokens(input: EstimateContextTokensInput): number {
  const chars = [input.systemRules, input.snapshot ?? "", ...input.messages, ...input.artifacts].join("\n").length;
  return Math.max(1, Math.ceil(chars / 2));
}

export function resolveContextBudgetMode(input: { estimate: number; maxInputTokens: number }): ContextBudgetMode {
  const ratio = input.estimate / input.maxInputTokens;
  if (ratio >= 0.7) return "compact_required";
  if (ratio >= 0.4) return "precompact_async";
  return "full";
}
```

- [ ] **Step 2：实现 SummaryValidator**

最小规则：

```ts
export type SummaryValidationArtifact = {
  id: string;
  title: string;
  kind: string;
  status: string;
  isApproved: boolean;
};

export type SummaryValidationResult = { status: "passed" | "failed"; errors: string[] };

export function validateSessionSummary(input: {
  summaryMarkdown: string;
  artifacts: SummaryValidationArtifact[];
  guardrails: string[];
}): SummaryValidationResult {
  const errors: string[] = [];
  const text = input.summaryMarkdown;
  const hasCompletionClaim = /已完成|可下载|最终完成|交付完成/.test(text);
  const hasUnfinishedArtifact = input.artifacts.some((artifact) => artifact.status !== "approved" || !artifact.isApproved);
  if (hasCompletionClaim && hasUnfinishedArtifact) {
    errors.push("摘要不能把未完成产物写成已完成。");
  }
  if (/长期偏好|以后都|每次都/.test(text) && !/待审批|未写入长期记忆/.test(text)) {
    errors.push("摘要不能把临时偏好直接写成长期记忆。");
  }
  return { status: errors.length > 0 ? "failed" : "passed", errors };
}
```

## 6. 风险与回退

### 6.1 风险

1. Prisma schema 改动影响本地 SQLite 数据。
2. `conversation-turn-service.ts` 已偏重，继续堆逻辑可能扩大屎山。
3. 压缩摘要如果校验不足，会丢失关键教学约束。
4. HumanGate actionId 若只做确定性字符串，安全性有限。
5. PromptCompiler 接入过深可能影响现有模型主 Agent 测试。

### 6.2 回退方式

1. 数据层改动只新增表和关系，不删除现有表字段。
2. ContextBuilder 先以兼容层接入，不删除旧 `conversationContext`。
3. SessionCompactor 先 deterministic/test 实现，真实模型压缩后续再接。
4. NodeContractRegistry 先文件化，不做 Admin 写入。
5. PlanGuard/HumanGate 先只阻断高风险外部 provider，不影响普通聊天。
