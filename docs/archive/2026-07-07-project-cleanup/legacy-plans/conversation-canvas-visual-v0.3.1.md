# v0.3.1 中央对话画布主视觉规划

## 1. 第一性原理

v0.3 降低了信息密度，但中央主对话区变得过于“白纸化”。教师视线需要一个稳定的主视觉承托：知道哪里是对话画布，哪里是当前任务，哪里可以继续输入。

本轮目标不是增加装饰，而是给中央对话条和任务流加底色层级。

成功标准：

- 中央主区有明确浅灰画布底，不再像元素漂在白纸上。
- 对话条有轻微底色，用户和 AI 消息仍然克制。
- 当前行动和当前产物在同一个视觉系统里，不新增彩色噪点。
- 左侧项目栏和右侧产物栏不改结构。
- 字号仍保持 12px、14px、16px 三档。

## 2. 参考与复用

参考 Codex 类产品界面：主内容区通常不是纯白裸排，而是有轻底板、细边界和输入框锚点。复用现有 React/Tailwind 组件，不引入新 UI 库。

## 3. 实现方式

- `ConversationWorkbench`：在滚动内容内部增加 `conversation-canvas` 浅灰底画布。
- `ChatTranscript`：消息正文从纯白卡片改为浅灰/白底的对话条，减少漂浮感。
- `GenerationPanel`：继承画布底色，当前行动使用白底轻边框，避免同一屏里灰色层级冲突。
- `PromptComposer`：保留当前轻阴影，作为底部输入锚点。

## 4. 验证

- `npm run build` 通过。
- 桌面 1440px 截图中，中间主区有明确视觉底。
- 390px 无横向溢出。
- 右侧节点点击详情仍可用。
