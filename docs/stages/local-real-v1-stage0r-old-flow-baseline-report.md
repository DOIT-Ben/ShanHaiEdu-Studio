## 1. 结论

固定教师任务已经从项目创建、对话规划、逐节点确认一路运行到真实阻断点，`0R-06` 的旧流程对照基线已建立。这里的“基线完成”不代表交付成功：当前旧流程没有生成可编辑 PPTX、视频资产图、分镜视频片段、最终视频或材料包，不能授课，也不能进入邀请制发布。

本轮证明了五件事：

1. Main Agent 能调用真实图片 Provider，并能在 PPTX 失败后接受显式改道。
2. 需求、教案、PPT 大纲、PPT 设计、知识锚点、创意、脚本、分镜和资产说明仍大量依赖弱结构草稿；上游丢失的课题和课时会持续污染下游。
3. 当前质量门主要判断文件存在和 MIME，不会阻止教学内容错误、占位符、泛化分镜或错误数学语义继续下传。
4. `asset_image_generate` 的失败没有形成 `GenerationJob`，一次受控自然语言重试也没有真正重启 Provider；恢复、幂等和成本台账不完整。
5. 缺少视频资产图时，系统没有伪造后续视频，但阻断文案错误地要求“已确认的分镜视频片段”，形成循环前置。

因此，Stage 1A 不能只调提示词。必须先完成执行身份、写租约、意图版本、幂等和 Provider Job 恢复；随后用节点合同和质量决策修复内容质量。

## 5. 真实图片检查

文件：

```text
.playwright-cli\cmrgpacsj00033oezlaguecfd-1783796318192-classroom-visual.png
```

画面为一名成人和一名学生观察两杯果汁，杯体直接出现 `75%`、`50%`，左侧保留大面积白色文字区。

优点：

- PNG 有效，尺寸和清晰度足够做课件底图。
- 主体集中，留白明确，课件排版空间充足。
- 百分数字样可读。

问题：

- 没有说明百分数对应的单位整体，容易把“杯中液面比例”和标签值混为既定答案。
- 人物与中国六年级课堂语境不一致。
- 画面没有形成具体可执行的观察任务。
- 数字被烘焙进图片，不可编辑，也未经过数学语义 Validator。

判定：`preview_only`；不能直接作为最终课堂视觉。

## 7. Agent 与交互行为

### 有效行为

- 显式“跳过当前节点”后能把 PPTX 改回待推进，并转向图片和视频链路。
- 视频资产图失败后，能读取失败 observation，遵守“不回退、不重做上游、停止重试”。
- 缺少资产图时没有调用视频 Provider，也没有伪造片段或成片。
- 已批准上游没有在受控重试中被覆盖。

### 失败行为

- 普通自然语言改道第一次被忽略，仍推荐生成 PPTX。
- 快捷回复需要“点击推荐 + 再点发送”，重复增加操作。
- 节点“已完成”和 Artifact“待确认”同时出现，状态语义冲突。
- 隐藏教材证据、导入视频方案由系统静默生成，教师不知道为什么必须批准。
- 上游明确输入在 deterministic draft 中丢失，随后污染所有视频文本节点。
- `asset_image_generate` 失败不进入 GenerationJob，Provider 恢复和成本台账断裂。
- 缺前置提示把“分镜视频片段”列为生成片段计划的前置，形成循环依赖文案。

## 8. 根因与 Stage 1 输入

| 根因 | 直接后果 | 下一阶段必须处理 |
|---|---|---|
| 教师原始意图没有形成不可丢失的结构化约束 | 40 分钟、苏教版、课题、交付范围丢失 | IntentVersion + Requirement Contract + prevalidate |
| 文本节点使用泛化 deterministic draft | PPT/视频只有模板结构 | 节点 Contract、专业 Prompt/Skill、Critic 和 QualityDecision |
| Provider 通道和 Job 台账不统一 | smoke 成功但应用 PPTX 失败；资产图失败无 Job | 所有外部副作用统一经 Provider Job/ToolRouter，支持恢复和幂等 |
| 质量门只查容器 | 错误图片和占位脚本被标成基础通过 | Artifact Truth 与 Delivery Effect Truth 分离 |
| HumanGate 与计划状态分裂 | “节点已完成”但成果待确认 | 统一状态机和教师可理解的状态文案 |
| Replan 依赖特定措辞 | 普通改道被忽略 | 意图版本栅栏、受控 ReAct 和最小消歧 |
| 能力依赖提示由错误集合拼装 | 出现循环前置 | ContractPreValidator 返回精确缺失项和修复动作 |
