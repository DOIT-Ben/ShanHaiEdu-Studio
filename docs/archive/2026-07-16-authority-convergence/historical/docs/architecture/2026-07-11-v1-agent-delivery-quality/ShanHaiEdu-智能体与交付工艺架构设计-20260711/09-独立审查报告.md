## 3. 最新工作树专项审查

### 3.1 已确认的有益变化

- pending plan 已出现 `superseded` 生命周期，改道后旧计划可退出活动状态。
- PlanGuard 开始以 ToolRegistry 的 side effect/HumanGate 定义为权威，不再让模型自行决定确认策略。
- policy block 不再污染工具失败预算。

这些方向应保留。

### 3.2 仍未闭合的 M72 问题

- M72 文件仍未提交，不能当稳定基线。
- Resolver 类型声明含 clarify/cancel/revise，但实际主要实现 confirm/switch/ordinary。
- Resolver 当前仍能用正则覆盖 Main Agent capability。
- ActionOffer 设计宣称绑定 planVersion/意图指纹，但当前 actionId 仍未覆盖完整版本、scope、inputHash 和 expiry 语义。
- internal tool 直执行已经放开，但 ContractPre/PostValidator 仍未进入主链。
- 完整交付成功后依然通过固定 delivery plan 计算下一步，不是 observation 驱动 Replan。

因此正确顺序不是继续扩大正则和直执行，而是先冻结 `AgentIntent / ActionOffer v2 / SideEffectPolicy`，再把 Contract 和最小 Replan 接到所有入口。

## 4. 两个 P0 为什么不会压制智能体能力

### ProjectExecutionLease

租约只限制会改变项目事实的动作，不限制 Main Agent 思考、Director/Critic 推理、普通对话、路径比较和只读验证。它约束的是并发写、重复付费和迟到覆盖，不约束创意。

### FinalDeliveryGate

最终门只回答“这组当前产物能否作为最终材料包发布”，不决定风格、模型、样张数、镜头数或叙事方案。required roles 由 DeliveryProfile 决定；视频只通过课程锚点回接教案，不被强迫读取 PPT 或复用 PPT 资产。

两者都是执行/交付边界，不是第二个 workflow planner。

## 5. 必须保留的智能体自由度

- 教师可以按需从教案、PPT、图片或视频能力切入。
- Main Agent 可以基于 observation 换路径、补最小前置、局部返修、暂停或追问。
- Director 可以按内容决定样张覆盖、镜头数量、参考策略和必要的关联变量联动。
- Profile 可以表达立体插画、板书、数据图表、纪录片等差异。
- Fast/Preview 可用于探索和快速反馈，只是不能冒充最终交付。

不可开放的是伪造事实、绕过安全、使用陈旧版本、重复外部调用、并发写乱状态和用模型自评替代真实证据。

## 6. 实施前验收矩阵

| 场景 | 预期结果 |
|---|---|
| 同项目两个状态推进请求同时到达 | 只一个获得 lease；同幂等请求复用，不重复 Provider 调用 |
| 不同项目同时生成 | 允许并发 |
| lease 过期后旧 worker 回写 | fencingToken 校验拒绝 |
| HumanGate 或 PreValidator 未通过 | 不占长租约，不调用 Provider |
| Provider 已有 taskId 后进程恢复 | 继续 poll/download，不重新 submit |
| Fast Coze PPT 请求最终包 | `FINAL_PREVIEW_ONLY_PATH` |
| Short Preview 视频请求最终包 | `FINAL_PREVIEW_ONLY_PATH` |
| lesson 或上游选择改版 | 旧报告、HumanDecision、GateReport 和最终包 stale |
| preflight 后 sourceRefs 变化 | `FINAL_GATE_INPUT_CHANGED`，不发布包 |
| ZIP 条目与 manifest/hash 不一致 | post gate block |
| PPT 缺页、重复 pageId 或范围聚合 | deterministic ValidationReport failed |
| 视频必需参考图未真实传 Provider | deterministic ValidationReport failed |
| Critic 给高分但 Validator failed | QualityDecision 必须 block |
| 单页/单镜头 major | 只路由到责任 page/shot/asset/track 返修 |
| 复合否定表达改道 | Main Agent/clarification 处理，Resolver 不按首个正则强制选能力 |
