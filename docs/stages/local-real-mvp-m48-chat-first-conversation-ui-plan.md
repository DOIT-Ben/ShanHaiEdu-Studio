# 本地真实 MVP M48 Chat-first 对话界面规划

日期：2026-07-08

## 目标

把教师工作台第一屏收敛成接近模型聊天产品的对话体验：

- 用户消息右侧气泡，AI 回复左侧气泡。
- 中间对话区成为第一视觉层，不再被聊天内时间线和常驻产物栏抢注意力。
- 输入框保持底部中心位置，回车、发送按钮、附件引用继续走 M47 的真实 API 链路。
- 发送后立即显示用户气泡，并在 AI 回复期间给出等待状态，避免“点了没反应”和并发乱序。
- 产物和流程仍可进入，但默认作为二级入口，不冒充纯聊天，也不压住聊天。

## 当前问题

- `ConversationWorkbench` 在对话内容旁挂 `ConversationNavigator`，页面像流程时间线，不像连续聊天。
- `ChatTranscript` 的 assistant 回复是无边界文本块，用户难以分辨“谁说了什么”。
- `MediaWorkbench` 桌面端常驻 `ArtifactRail`，右侧节点抢走主视觉，让用户感觉还停留在工作流原型。
- 空状态文案偏任务说明，不像可以直接开始聊天的模型入口。
- 模型回复较慢时，用户消息不会先出现在对话区，连续发送还可能造成顺序感混乱。

## 范围

- 修改 `ChatTranscript.tsx`：补齐 AI 与用户的稳定对话气泡、角色标签、换行保留、低噪声复制按钮。
- 修改 `ConversationWorkbench.tsx`：移除聊天内 `ConversationNavigator`，调整滚动内容宽度和空状态。
- 修改 `MediaWorkbench.tsx`：桌面端不再默认常驻窄产物栏，改为顶部/移动一致的产物按钮打开 drawer；侧边详情面板仍保留。
- 修改 `useWorkbenchController.ts` 与 `PromptComposer.tsx`：增加发送等待状态，发送后乐观显示用户消息，回复期间阻止重复发送。
- 保留 `ProjectSidebar`、`StageProgress`、`ArtifactSidePanel`、`ArtifactDetailSheet` 和所有后端 API 合同。

## 非目标

- 不改服务端 conversation orchestrator。
- 不改 artifact 生成、确认、重做、下载和权限逻辑。
- 不做营销页或介绍页。
- 不把 deterministic draft 说成真实模型生成。
- 不做真实 PDF/OCR/二进制附件持久化。

## 成功标准

- 打开本地页面后，第一眼是对话，不是流程看板。
- 教师发送消息后，用户消息在右侧，AI 回复在左侧，并可清楚阅读多行内容。
- 教师点击发送后立即看到自己的气泡；AI 未回复前发送按钮不可重复触发。
- 桌面端没有常驻右侧产物 rail；产物通过“产物”按钮或详情面板按需打开。
- 空状态像聊天入口，能直接引导用户输入一句公开课需求。
- M47 的 Enter 发送、首发自动建项目、文本附件引用不回归。

## 风险与回退

- 风险：隐藏常驻产物栏后，产物发现性降低。控制方式：在顶部保留“产物”按钮，并继续支持详情 sheet。
- 风险：气泡化后长文阅读变窄。控制方式：AI 气泡最大宽度高于用户气泡，正文保留换行，长内容仍可在产物详情中阅读。
- 回退：如验收发现产物入口过弱，可恢复为可折叠窄 rail，但默认仍应保持关闭。
