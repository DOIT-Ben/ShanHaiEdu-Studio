# 后续整体分析问题清单

## 1. 整体分析目标

后续不能只回答“选 LangGraph 还是 Agents SDK”，而要形成一套完整的 ShanHaiEdu 智能体与交付质量总架构，并明确每一层的唯一权威源、数据契约和恢复语义。

## 2. 必须回答的架构问题

### A. Agent Runtime

1. 谁是唯一运行编排权威：LangGraph、OpenAI Agents SDK、Vercel Workflow SDK，还是保留自研？
2. Main Agent 是图中的 Planner 节点，还是一个可以调用专业 Agent 的 Manager？
3. ReAct 循环发生在主图、专业子图还是 Agent 内部？如何避免嵌套循环？
4. 最大步数、预算、超时和模型降级由谁控制？
5. Agent 输出什么结构，哪些内容只能由服务器补全？

### B. 状态与持久化

6. Graph State、Conversation State、Artifact State、Provider Job State 的明确边界是什么？
7. 哪个数据库是业务事实源，框架 Checkpoint 存什么、不存什么？
8. 如何保存 `runtimeVersion`、`graphVersion`、`stateSchemaVersion`、`promptVersion` 和 `contractVersion`？
9. 图升级后如何恢复旧 Run：兼容迁移、双版本执行、取消重跑还是人工处置？
10. 本地 SQLite 和生产数据库分别使用什么 Checkpointer？

### C. 打断、改道与恢复

11. 教师自然语言打断处于模型推理、Tool 执行、Provider 排队、Provider 生成、Artifact 审核时，各自如何处理？
12. Provider 不支持取消时，旧结果如何 quarantine、obsolete 或 stale？
13. ChangeSet 的结构是什么，ImpactResolver 如何计算下游失效范围？
14. 用户回跳后是恢复旧 Checkpoint，还是基于新 Artifact 版本重新 Replan？
15. 如何保证恢复不重复提交 Provider、不重复扣费、不重复批准？

### D. Tools 与专业智能体

16. 哪些能力封装为 Tool，哪些封装为 Skill，哪些是确定性 Validator？
17. PPT Director、Video Director、Critic 是子图、Agent-as-Tool 还是普通结构化调用？
18. 专业智能体能否直接执行有副作用 Tool，还是只能返回 Plan 给 Main Agent？
19. ToolRouter 如何继续执行权限、参数、Artifact 引用和 Provider 约束？
20. Tool Observation 如何统一表达成功、失败、待审批、可重试和局部质量缺陷？

### E. 交付质量

21. PPT 从叙事大纲、逐页设计、样张、全量生成到终检的每个 Gate 是什么？
22. 视频从课程锚点、创意主题、Beat、ShotSpec、资产、片段到组装的每个 Gate 是什么？
23. Rubric 是纯模型评分、确定性检查还是混合决策？
24. QualityDecision 如何选择 accept、repair、replan、ask_user、block？
25. FinalDeliveryGate 如何校验真实 PPTX 页数、视频时长、文件存在、血缘和批准状态？

### F. 交互、可观测性与治理

26. 前端需要哪些稳定事件：thinking、planning、tool_started、provider_waiting、reviewing、interrupted、resumed、completed？
27. 对话消息、图事件和业务事件如何去重及排序？
28. Trace 中哪些内容必须脱敏，哪些不允许发送到第三方平台？
29. 如何衡量框架化后的成功：完成率、恢复率、重复提交率、返修粒度、教师干预成本、交付质量？
30. 出现框架问题时如何切回现有 Runtime Adapter？

### G. 教师委托与信任

31. 哪个版本化对象代表教师当前委托，如何区分硬要求、软偏好、临时想法、系统默认和模型推断？
32. “只改这一页、其余别动”如何形成保护锁、影响预览、成本/时间增量和可撤销变更？
33. 教师可选择哪些自主度模式，哪些可逆动作默认自动执行，哪些动作必须合并确认？
34. Teacher Memory 如何记录来源、作用域、置信度、有效期、同意、修改和删除？
35. 系统如何向教师诚实说明能力边界、预计等待、成本区间、降级方案和人工接管点？

### H. 真实课堂、安全与产品真值

36. 完整公开课如何形成跨教案、PPT、视频、板书、互动和评价的统一 ClassroomRunSpec？
37. 如何验证目标 PowerPoint/WPS、字体、投影、音响、网络和离线备用环境？
38. 教材、学生照片/声音、校本材料和第三方素材在进入 Provider 前如何完成数据分级、脱敏、授权和版权检查？
39. 如何导入教师已有 PPT/PDF/DOCX/图片/视频，并在教师离线手改后进行差异合并而不覆盖人工修改？
40. “首次可授课”如何用多学科、多年级、真实教师修改时长、放弃率和课堂事故进行验证，而不是只依赖 Critic 分数？

| 交付物 | 目的 |
|---|---|
| 总架构决策 ADR | 确定唯一 Agent Runtime 和职责边界 |
