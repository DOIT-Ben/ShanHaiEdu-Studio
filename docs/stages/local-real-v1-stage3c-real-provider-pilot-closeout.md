## 4. Visual Review and Page Repair

The first full contact sheet was rejected at `page_04`: its AI scene carried generated unreadable text, violating the no-text image policy. The system persisted a failed critic review with the precise page locator and did not make the candidate final-eligible.

Only `scene_page_04` was submitted again to the real provider. The subsequent repair:

- replaced the page 4 scene asset and render;
- reused the other 11 page render references/hashes;
- rebuilt the aggregate PPTX, PDF, and contact sheet;
- passed visual re-inspection without generated text artifacts;
- wrote a fresh candidate digest and then an explicitly simulated integration review.

This is the required `Observe -> Plan -> Guard -> Act -> Observe -> Replan` behavior at page scope, not a blind full-deck rerun.

## 5. Contract Fix Discovered by the Pilot

The pilot exposed an approval bug: full-deck artifacts retain key-sample lineage, but repository approval incorrectly validated the key-sample set against the full-production manifest. These manifests intentionally have different request and asset digests.

`attachPptSampleApproval` now returns after validating a `PptFullDeckPackage`. The full package keeps sample lineage for audit, while its own candidate/package validation remains authoritative. A regression test covers a full-deck package that carries both full asset metadata and key-sample lineage.
