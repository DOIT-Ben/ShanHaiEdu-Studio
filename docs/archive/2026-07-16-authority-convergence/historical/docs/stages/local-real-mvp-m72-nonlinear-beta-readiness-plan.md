# Local Real MVP M72 非线性能力与团队内测收口计划

更新时间：2026-07-11

## 1. 目标

修复反馈面板状态跳变和消息评价缺少提示；根除自然语言确认与工具执行协议脱节；允许教师从教案、PPT、图片、视频等任意能力切入；完善密码账号入口、数据归属和双账号隔离证据；最终输出真实团队内测阻塞项。

## 2. 范围

1. 反馈弹窗使用稳定布局，快速补充区始终存在，只更新选项和引导文案。
2. 消息赞踩提供保存、切换和取消后的即时提示。
3. 引入统一对话控制决策：确认当前动作、切换能力、取消、修改、消歧和普通消息。
4. 低副作用内部能力允许按需执行；真实 provider、文件生成、外部写入和高成本调用保留 HumanGate。
5. 无活动 ActionOffer 的直接执行请求不得进入无解阻断，必须直接执行安全能力或创建可确认动作。
6. 复核并完善密码登录、最简注册/准入页面、owner/membership 隔离和历史项目绑定。
7. 不修改永久删除，不放松 PPTX、视频、最终包真实性和质量门禁。

## 3. 实施顺序

1. 先补失败测试：反馈稳定布局、评价提示、自然语言确认、直接能力切换、旧计划失效、双账号隔离。
2. 修反馈 UI 与评价状态提示。
3. 实现 ConversationControlResolver 和活动计划生命周期；把 policy 拒绝与工具失败预算分离。
4. 调整工具确认策略：安全内部工具直接执行，真实 provider/文件/外部副作用继续确认。
5. 完善认证页面和账号归属迁移脚本/服务，使用安全哈希写入指定账号。
6. 执行浏览器、API、双账号、真实工具和全量工程验收。

## 4. 风险与回退

- 风险：错误放开真实 provider 调用。控制：确认策略由服务端 ToolRegistry 判定，不由模型自行决定。
- 风险：自然语言误确认旧动作。控制：动作绑定项目、能力、消息、计划版本和意图指纹，改道后旧动作失效。
- 风险：历史数据误归属。控制：迁移前统计，事务内仅补 owner/membership，不删除数据；保留回退清单。
- 风险：公开注册扩大攻击面。控制：默认仍关闭公网自助注册；本地/明确配置下提供最简注册，团队内测优先管理员邀请。

## 5. 成功标准

- 反馈切换无整体跳变；评价有明确提示。
- 用户可直接要求任一能力，不被线性顺序机械阻断。
- 明确自然语言确认可执行已披露动作；模糊语义安全消歧。
- 双账号数据隔离通过真实请求验证，指定账号拥有历史数据。
- 类型检查、相关测试、全量测试、生产构建和桌面/390px 浏览器检查全部通过。
