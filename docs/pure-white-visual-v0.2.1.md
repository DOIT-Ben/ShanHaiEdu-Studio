# v0.2.1 纯白低噪声视觉收敛规划

## 1. 第一性原理

本轮核心不是增加功能，而是降低教师主界面的视觉噪声。用户明确要求接近 Codex 式工作台：纯白、浅灰、少色、少字号、少按钮存在感。界面必须让注意力回到中央对话和当前任务，左侧项目与右侧节点只承担导航和复用辅助。

可验证成功标准：

- 主界面远看只出现白、浅灰、黑灰文字三层；错误态才使用低饱和红。
- 不再出现黄、橙、青绿大面积强调或彩色渐变缩略块。
- 按钮不使用突兀深色填充，发送和确认也保持轻量。
- 字号层级收敛到正文 14px、说明 12px、模块标题 15-16px。
- 右侧节点仍是收起的线性产物串，hover 预览和点击详情不丢。
- 不出现 emoji 或教师不可理解的工程词。

## 2. 复用与参考

复用现有 Next.js、Tailwind、Radix、lucide 和已拆分业务组件，不引入新 UI 库。参考用户提供的 Codex 截图，只取产品工作台结构特征：

- 左侧浅灰项目栏，当前项用低对比灰底。
- 中央纯白阅读/对话区，内容卡片边框很轻。
- 底部输入框是主要视觉锚点，但不使用强色按钮。
- 状态与操作靠文字、边框、浅灰底表达，不靠多色徽标。

`gpt-taste` 本轮只采用可兼容约束：专业间距、按钮可读性、禁止 emoji、避免廉价视觉噪声。其营销页 AIDA、hero、GSAP 动效要求与本产品工作台目标冲突，不采用。

## 3. 复用、适配和必要自研

复用：

- `Button`、`Badge`、`Sheet`、`Popover`、`ScrollArea`、`Textarea` 基础组件。
- 现有 `ProjectSidebar`、`ConversationWorkbench`、`ArtifactRail` 拆分架构。
- 现有 mock 项目、消息、产物数据结构。

适配：

- 全局 token 统一为白、浅灰、深灰、低饱和红。
- `Button` 默认改为浅灰/白底边框，不再强填充。
- `Badge` 非错误状态全部改为灰色，状态差异交给文字。
- 生成卡、阶段条、会话头像、右侧节点、详情缩略图去掉彩色强调。

必要自研：

- 无新增组件。只在现有组件内收敛样式，避免重构过度。

## 4. 开发方案、风险和验证

开发方案：

1. 更新全局 token 与基础按钮、徽标、浮层阴影。
2. 收敛左侧项目栏：浅灰栏、灰底选中、品牌和状态低调化。
3. 收敛中央会话区：灰色头像、轻阶段条、白底生成卡、浅灰进度。
4. 收敛右侧节点串：灰色节点、细线串联、轻浮层预览。
5. 收敛详情抽屉：灰色缩略图、灰色 tab、轻按钮。

风险：

- 颜色过少可能导致状态识别变弱，因此保留状态文字；错误态保留低饱和红。
- 按钮全部浅化后主操作可能不明显，因此发送/确认用稍深浅灰底和位置强化，而不是颜色强化。
- 极简化不能破坏节点 hover、复制、作为输入、详情抽屉这些核心交互。

验证标准：

- `npm run build` 通过。
- 桌面页面没有黄色、橙色、青绿大面积视觉块。
- 桌面 1440px 左中右结构仍清晰，中央对话是视觉中心。
- 390px 页面不横向溢出，项目和产物抽屉仍可用。
- 控制台无前端错误。
