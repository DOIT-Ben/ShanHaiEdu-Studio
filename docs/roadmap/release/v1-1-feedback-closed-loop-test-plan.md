# V1.1 对话 Runtime 与反馈闭环验收计划

日期：2026-07-13

状态：draft；实施前由各阶段补充精确测试文件与夹具

关联规划：`docs\roadmap\release\v1-1-feedback-closed-loop-plan.md`

## 1. 验收原则

- 原有反馈中心能力先做回归，再验收新增闭环。
- 原有V1消息、任务、HumanGate、Artifact和下载路径先做兼容回归，再切换assistant-ui正式入口。
- 自动化证明合同和权限，真实浏览器证明教师能用，生产演练证明数据不会因发布或回退丢失。
- 任何 mock、前端定时器或管理员手工改库都不能代替真实状态推进。
- V1.1 必须由至少两名受邀教师完成真实任务与反馈闭环；外部验收者只审核结果，不代替产品流程修改状态。
- assistant-ui组件显示成功不能证明业务成功；所有Tool、HumanGate、Quality和Artifact状态必须追溯到服务端权威事实。

## 3. 浏览器验收

### 3.1 桌面 1366x768

- 欢迎页进入项目，对话区、成果区和反馈入口层级清楚。
- assistant-ui线程正确显示旧消息和新MessagePart；文本增量、结构块、Tool状态、成果引用、HumanGate和错误恢复无跳动、重复或裸Markdown。
- 编辑、重试、分支和队列只在服务端允许时出现；已发生真实副作用的历史消息不能通过通用重试直接重放。
- 当前计划首次以逐行计划视图展示；稳定执行与收尾可转为紧凑进度视图，教师可以随时展开或收起且不改变任务状态。
- 完成绿、进行中蓝、未开始灰、等待确认琥珀和失败红同时具有文字/图标语义；等待确认、失败、Replan 和完成不遮挡输入。
- 真实耗时随状态恢复一致；无可信统计时不出现剩余时间或伪倒计时。
- 模拟真实任务重试耗尽后出现安全错误条；“反馈问题”打开已预填表单，诊断摘要可查看/关闭，关闭或取消时不创建反馈。
- `Ctrl/Cmd+K` 打开搜索与快捷操作；无查询、输入筛选、分类切换、无结果和失败态均可操作。
- PPT当前页、教案当前段、视频镜头/时间段均能打开已预填目标的反馈弹窗。
- 管理员能分诊、归因、关联版本、进入验证和解决；教师能看到安全回告。
- 键盘 Tab、Enter、Space、Escape、focus-visible 和 reduced motion 正常。

### 3.2 窄屏 390x844

- 项目、成果使用抽屉，中央对话、计划和输入无横向溢出。
- 流式长段落、列表、表格、Tool状态、HumanGate和Artifact引用不遮挡输入，长单词和教师文本可换行。
- 长计划、10步以上、等待确认和失败原因内部滚动，不遮挡发送。
- 紧凑进度视图标题与当前活动可换行，分段按业务里程碑聚合，不出现不可辨认的细碎色条。
- 活动流已完成阶段默认折叠，当前任务和等待确认保持可见；长任务滚动不遮挡底部计划和输入。
- 错误条按钮和已预填反馈面板不遮挡软键盘；搜索使用近全屏Sheet，输入固定、结果独立滚动。
- 反馈分类、截图预览、期望图、我的反馈和补充信息流程可完成。
- 视频时间段、PPT页和教案段落 locator 文案换行正常。

## 4. 端到端闭环用例

### 用例0：对话Runtime迁移与恢复

1. 使用包含旧文本、计划、快捷回复和Artifact引用的V1项目进入assistant-ui测试入口。
2. 对比旧UI与新Runtime的消息数量、顺序、正文、角色、成果版本和活动状态。
3. 发起一个无需HumanGate的长任务，观察真实文本和活动事件流；中途断网、刷新并重新进入项目。
4. 客户端从服务端快照与最后sequence恢复，不重复消息、Tool调用、费用或状态提升。
5. 触发一个真实PendingDecision，分别使用按钮和自然语言继续，确认只解析同一个action。
6. 切回旧UI回退入口，消息和业务状态一致；再切回assistant-ui时不产生双写。

### 用例 A：页面或流程问题

1. 教师甲遇到真实任务错误，从错误条点击“反馈问题”。
2. 系统预填安全摘要、当时任务和期望结果，并展示可关闭的脱敏诊断信息；教师补充截图后提交。
3. 系统保存并返回回执，管理员看到合法错误现场和diagnostic fingerprint。
4. 管理员分诊、归因、关联需求和目标版本。
5. 修复后进入 verifying，附回归证据。
6. 标记 resolved 并向教师甲回告。
7. 教师甲可看到版本与结果，教师乙不可读取。

### 用例 B：生成产物局部问题

1. 教师乙对 PPT 某页或视频某个 shot/time range 提交反馈。
2. locator 唯一绑定当前产物版本，不指向旧版本。
3. 管理员归因到 Agent、Prompt、Tool、Provider、质量门禁或前端中的实际责任层。
4. 完成局部返修和回归；未受影响产物不重跑。
5. 新版本解决后回告；若教师选择“问题仍存在”，case 重新打开并保留历史。

## 5. 安全与隐私用例

- 普通教师读取他人反馈、附件、case和回告均返回拒绝或不泄露存在性的 404。
- 管理员内部字段不会出现在教师API、页面、导出给教师的内容或错误消息。
- 伪造跨项目 messageId、artifactId、versionId、planId、shotId 和 pageId 被服务端拒绝。
- 用户描述不会进入服务端路径、日志标签、指标标签或未转义 CSV 公式。
- Prompt、Provider、模型名称、token、请求头、密钥和本地路径不进入教师页面。
- 诊断包不包含完整对话、Prompt、请求/响应正文、Cookie、凭据、签名URL、原始堆栈或未授权截图；关闭诊断开关后服务端不接收诊断字段。
- 教师甲的搜索结果不包含教师乙未共享的项目、任务、成果和反馈；管理员入口按角色实时过滤。
- 活动流不包含模型思维链、原始Prompt、命令、API、Provider、路径和调试日志。
- assistant-ui自定义Renderer拒绝脚本、事件处理器、危险URL和伪造系统状态；用户Markdown/HTML不能生成HumanGate按钮或完成标识。
- AG-UI Raw事件、完整Tool参数和未列入白名单的Custom事件不进入教师客户端。

## 6. 生产与回退验收

1. 发布前备份 SQLite、反馈附件和必要配置。
2. 在隔离副本验证旧数据库加法升级和重复初始化。
3. 生产重启后读取一条旧反馈、一条新 case、附件和教师回告。
4. release 回滚继续挂载共享数据卷，不删除新表和附件。
5. 如果旧代码不能理解新业务字段，必须回退到兼容版本；不得通过删表恢复。
6. 恢复备份后，回执、case关系、事件顺序和附件 hash 一致。
7. assistant-ui功能开关回退后仍读取同一消息和业务状态，不删除新字段，不重放Tool。

## 7. V1.1 发布门

只有以下条件同时满足才允许发布：

- 所有定向测试、`npm test`、TypeScript、Prisma validate、生产构建和 diff check 通过。
- 1366x768 与 390x844 浏览器验收通过，无 P0/P1 视觉或交互阻塞。
- 两用户隔离、长任务活动流、授权搜索和两条真实反馈闭环用例通过，其中一条来自真实错误现场。
- assistant-ui为正式唯一对话入口；历史消息兼容、真流式、断线恢复、受控回调、旧UI回退和无双写证据通过。
- 重启、备份恢复、release 回滚和公开注册关闭复核通过。
- 教师界面无工程词、敏感字段和伪造状态。
- 已知问题、回退方式、发布说明和责任人已记录。

任何一项缺失时只能标记为阶段实现完成，不能标记 V1.1 已上线。
