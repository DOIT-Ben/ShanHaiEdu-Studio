# Local Real MVP M74 品牌化认证入口计划

更新时间：2026-07-11

## 1. 目标

将现有基础密码认证表单升级为山海教育品牌化登录/创建账号入口，使小学教师在进入工作台前清楚理解产品用途，并在桌面与移动端获得安静、可信、可访问的认证体验。

## 2. 范围与边界

- 只修改 RQ-017、M74 plan/test-plan、`PasswordAuthGate` 和直接相关认证合同测试。
- 复用 `public/brand/auth-teacher-cover.png`、`public/brand/shanhai-education-logo.png`、Next Image、Tailwind、Button 和 Lucide，不新增依赖。
- 保持 `onLogin` / `onRegister` 参数合同与账号、显示名（注册）、密码的字段顺序。
- 保持运行时 `registrationEnabled` 门禁；关闭注册时不显示创建账号入口。
- 不修改认证服务、API、会话、数据结构、对话、产物或反馈。

## 3. 设计与实施

1. 桌面 `lg` 起采用 54/46 双栏；左侧用 `next/image` 的 `fill` + `object-cover` 呈现沉浸封面，并在底部叠加纯深蓝透明遮罩与三层具体品牌文案。
2. 右侧为页面自身的认证区域而非悬浮卡片，设置合理内容最大宽度；顶部使用 84px 独立产品图标与“山海课伴 / 山海教育 · AI 备课工作台”的紧凑品牌签名，避免带字企业 Logo 缩小后识别不清，也避免母品牌与产品名层级重复。
3. 标题按有效模式动态显示“欢迎回来”或“创建你的教师账号”，副文案明确登录/创建后回到自己的备课项目。
4. 模式切换使用安静的分段按钮，补齐 `aria-pressed`、禁用态与清晰焦点；以 `registrationEnabled && mode === register` 作为有效注册模式，避免门禁变化后残留注册提交。
5. 保留现有表单行为，补充账号、显示名和密码 placeholder；注册密码提示至少 8 位，不出现后端工程词。
6. 增加真实、轻量的可见范围说明，不声称云端加密、学校认证或不存在功能。
7. `lg` 以下改用 160px 横向图片头；认证内容紧凑排列，输入框和提交按钮至少 44px，避免横向溢出；桌面 1366×768 不依赖页面滚动。

## 4. 风险与回退

- 风险：现有认证组件和测试有未提交 M72 改动。控制：只做增量编辑，保留运行时注册门禁和账号字段合同。
- 风险：静态图片影响首屏布局。控制：桌面封面使用 `fill`，移动图头固定高度并使用 `sizes`；Logo 使用明确尺寸。
- 风险：模式门禁关闭时仍呈现注册状态。控制：所有标题、字段、自动填充和提交分支统一读取有效注册模式。
- 风险：1366×768 内容溢出。控制：桌面区不使用额外纵向大间距，表单宽度限制在 420px 内。
- 回退：可独立回退 M74 文档、组件视觉和合同测试，不涉及数据库迁移或认证后端。

## 5. 成功标准

- 桌面双栏与移动品牌图头符合尺寸和响应式边界，无卡片悬浮、渐变大屏或玻璃拟态。
- 品牌资产、具体定位文案、动态标题、可信说明、注册门禁和字段行为完整。
- 定向测试、认证测试、`tsc --noEmit`、单 worker 全量测试、生产构建和 `git diff --check` 通过。
- 浏览器结果不在本阶段代报，由主代理实际执行后给出结论。
