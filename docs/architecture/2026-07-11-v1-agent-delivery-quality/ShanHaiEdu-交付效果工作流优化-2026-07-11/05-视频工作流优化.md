# 视频工作流优化

## 1. 工作流目标

导入视频先是一条能独立吸引学生的短片，再通过唯一必要的“课程锚点”回到课堂任务；它不是教案摘要，也不是 PPT 页脚本的动态版。

## 2. Agent 可选黄金路径

下列是复杂多镜头导入视频的参考路线，不是每个请求都必须从头执行的固定 DAG。Main Agent 可以根据已批准创意、时长、连续性需求、预算和 Observation 跳过已满足节点、拆分镜头、并行安全生成或回退到局部修复。

```text
Lesson Plan
  -> Course Anchor
  -> 三类九套独立创意
  -> Concept Selection Set
  -> Beat Sheet
  -> ShotSpec Set
  -> Asset Bible
  -> Asset Generation
  -> Segment Requests
  -> Segment Generation
  -> Segment QA
  -> FFmpeg / HyperFrames Timeline
  -> Final Video QA
  -> Classroom Landing Review
```

不可绕过的只有：版本化课程锚点、真实生成或确定性制作的媒体、逐片段/成片技术真值、课堂边界和最终交付审计。候选数量、是否需要完整 Asset Bible、Beat Sheet 与 ShotSpec 是否分开，由 Agent 根据任务复杂度决定。

## 3. 课程锚点

课程锚点是导入视频与教案之间唯一必需的对齐点：它把独立视频主题拉回课堂学习任务，但不提前讲知识点结论。

结构：

- `anchor_trigger`：视频中引发课堂问题的事件。
- `classroom_question`：教师接住视频的第一问。
- `do_not_explain`：视频绝不能提前讲出的结论。
- `handoff_moment`：视频结束后进入哪一个课堂动作。

## 4. 三类九套创意

当教师尚未给出明确创意、需要探索时，默认生成三类、每类三案：

1. 叙事情境类：角色遭遇问题、悬念或误会。
2. 视觉奇观类：尺度、变形、快速变化或意外组合。
3. 真实生活挑战类：生活现场、任务委托或观察谜题。

进入候选探索模式时，每案必须输出：方案编号、独立主题、吸睛点、新奇特标签、一句话故事、课程锚点、课堂落点问题、不提前讲解内容、推荐分、推荐理由和适配风险。教师已经给出可执行创意时，可以直接形成单案选择记录，不强制补做九案。

选择集必须锁定：`selection_mode`、`selected_design_ids`、`primary_design_id`、`selection_reason`、`downstream_generation_mode`。下游只能读取选择集，不得回到九套候选自行改选。

## 5. Beat Sheet

每个 beat 先描述观众体验，而不是页面布局：

- `concept`：这一段的视觉世界和叙事作用。
- `viewer_feeling`：好奇、紧张、惊讶、发现或停顿。
- `choreography_verbs`：主体和元素具体如何移动。
- `visual_focus_1/2`：至少两个视线落点。
- `depth_layers`：前景、中景、背景。
- `transition`：与下一 beat 的关系。
- `audio_cue`：旁白、环境音、音乐或留白。
- `duration_budget`：进入、呼吸、解决三个阶段。

## 6. ShotSpec

视频 Provider 每次只消费一个 ShotSpec：

- 主体、动作、场景和时间。
- 景别、机位、镜头运动和速度。
- 起始帧状态、结束帧状态。
- 角色/场景/道具资产引用。
- 旁白、字幕和必要环境音。
- 连续性键：服装、位置、视线、光线、色彩、道具状态。
- 负面约束：文字、品牌、额外人物、错误数量、畸变等。
- 课程锚点关系。
- 镜头独有验收条件。

禁止只传“明亮温暖的小学课堂导入视频”一类笼统描述。

## 7. Asset Bible

至少包含：

- 风格基准图。
- 主角色正面/侧面/表情参考。
- 核心场景全景与关键区域。
- 道具和数学对象参考。
- 每个 Shot 的首帧/末帧关键帧。
- 色板、光线和材质说明。
- 不允许变化的连续性键。

资产图必须来自视频资产分支，禁止回退使用 PPT 渲染图或联系表作为参考。

## 8. 片段生成策略

- 一镜头一请求，保留 shot_id。
- Prompt 顺序：主体 -> 动作 -> 场景 -> 构图/机位 -> 镜头运动 -> 风格/光线 -> 音频 -> 负面约束。
- 同一角色始终复用参考图和 continuity key。
- 需要精确数字、数量、字幕或公式时，用 HyperFrames/本地层叠加，不依赖生成视频内文字。
- 失败重生只改变一个主变量，记录变化原因。

## 9. 正规合成

禁止 `Buffer.concat(MP4[])`。推荐两条路线：

### 同参数片段

使用 FFmpeg concat demuxer，要求相同 codec、time base、分辨率、帧率和音频参数；重新生成时间戳并验证总时长。

### 参数不一致或需要字幕/混音

先统一解码、缩放、帧率、采样率和声道，再使用 concat filter 重编码；最后加字幕、旁白、BGM、响度标准化和 `+faststart`。

## 10. 质量门

### 片段级

- 可完整解码，时长在计划容差内。
- 首尾关键帧与 ShotSpec 一致。
- 主体、动作、数量、角色和场景正确。
- 无黑帧、冻结、严重闪烁、异常形变。
- 字幕或本地数学层不遮挡主体。

### 成片级

- 总时长、分辨率、帧率、H.264/AAC 和 `faststart` 合格。
- 音频可懂，旁白与 BGM 不争夺；无削波和过低响度。
- 字幕与语音同步，并包含理解所需的非语言声音。
- 镜头连接无重复帧、跳轴、角色漂移和时间戳断裂。
- 节奏有 build/breathe/resolve，不把长静帧误当作“高级留白”。
- 结尾形成问题和停顿，不给出课堂答案。

## 11. 镜头级返工

- 动作失败：仅重生该 Shot。
- 连续性失败：重生当前 Shot，使用上一 Shot 的末帧作为额外参考。
- 字幕/数字失败：改本地确定性层，不重生底层视频。
- 音频失败：重做旁白/混音，不动视频画面。
- 节奏失败：调整时间轴或剪辑点；只有素材不足时才重生片段。
- 课程锚点失败：修改结尾 beat 和课堂落点，不把整条视频改写成教案讲解。
