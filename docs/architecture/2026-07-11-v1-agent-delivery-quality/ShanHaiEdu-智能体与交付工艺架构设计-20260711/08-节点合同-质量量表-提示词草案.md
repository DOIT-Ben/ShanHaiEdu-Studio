# 08 节点合同、质量量表与提示词评审稿

> 状态：实施前评审候选，不是项目代码，也不是已发布 Schema。本文把职责和判定语义定到可实现程度；字段命名、TTL 和初始阈值仍需在正式规格及教师样本校准后冻结。

## 1. 统一术语

### 1.1 优先级与问题严重度分离

- `releasePriority=P0|P1|P2|P3`：表示路线实施优先级。
- `severity=blocker|major|minor`：表示某一产物质量问题的严重度。

禁止再用 P0/P1 同时表示“实施优先级”和“质量问题级别”。

| 严重度 | 定义 | 机器结果 | 人工权限 |
|---|---|---|---|
| `blocker` | 事实错误、安全违规、真实文件/血缘失败、核心内容不可用或无证据可证明 | `block` | 不可豁免、不可降级 |
| `major` | 明显影响教学、理解、可读性或连续性，但可以定位返修 | 至少 `repair` | 可以保留预览，不能批准最终交付 |
| `minor` | 不影响事实、核心理解和课堂使用的局部瑕疵 | 不单独阻止 `pass` | 教师可在最终版本上接受 |

### 1.2 路径身份与当前资格分离

```yaml
ProductionIdentity:
  productionPath: string
  qualityTier: preview | quality
  eligibilityCeiling: preview_only | unit_candidate | final_eligible

ArtifactDeliveryState:
  deliveryEligibility: not_eligible | preview_only | unit_candidate | final_candidate | final_eligible
```

路径决定上限，当前质量和人工决定决定状态。字段修改不能升级路径身份；升级必须生成具有新 `inputHash/lineageHash` 的新 Artifact。

| productionPath | qualityTier | eligibilityCeiling | 说明 |
|---|---|---|---|
| `ppt_fast_coze` | `preview` | `preview_only` | 快速预览，不得进入最终包 |
| `ppt_quality_asset_assembly` | `quality` | `final_eligible` | 通过完整质量链后可成为最终 PPT |
| `video_short_preview` | `preview` | `preview_only` | 单镜头/短预览，不代表完整导入视频 |
| `video_full_intro` | `quality` | `final_eligible` | 完整时间线通过后可成为最终视频 |

QualityDecision `pass` 最多把 Quality/Full 产物推进到 `final_candidate`；只有绑定同一版本、digest、inputHash 和质量决定的 `HumanDecision(approve_final)` 才能推进到 `final_eligible`。FinalDeliveryGate 只接收 `final_eligible`。

设计约束：

- Contract 只放不可绕过的边界，不固定模型、固定镜头数、固定样张数或视觉风格。

## 3. Agent-as-Tool 运输合同

当前 `internal_capability` 只支持固定 Capability 到 Markdown Artifact。Director/Critic 需要独立通用协议：

```yaml
AgentToolDefinition:
  toolId: ppt_director.plan_or_repair
  agentProfileId: ppt_director.v1
  adapterKind: agent
  inputContractRef: contract://agent-tool/ppt-director-input/1.0.0
  outputContractRef: contract://agent-tool/ppt-director-output/1.0.0
  allowedToolClasses: [read_only]
  executionSideEffectLevel: none
  resultPersistencePolicy: report_artifact
  persistenceSideEffectLevel: artifact_write
  persistenceLeaseMode: short_persist
  requiresHumanGate: false

AgentToolInvocation:
  invocationId: string
  projectId: string
  runId: string
  sourceMessageId: string
  worldStateVersion: integer
  stage: string
  targetLocators: [TargetLocator]
  trustedArtifactRefs: [VersionedArtifactRef]
  reportRefs: [VersionedReportRef]
  skillSliceRef: string
  profileRef: string
  contractRef: string
  inputHash: string

AgentToolResult:
  status: succeeded | needs_input | failed | inconclusive
  structuredOutput: object
  reportArtifactDraft: object
  observation: object
  provenance:
    runtimeProfile: string
    model: string
    promptVersion: string
    skillVersion: string
    contractVersion: string
    inputHash: string
```

规则：

- Director/Critic 是 Tool，仍通过 ToolRouter 或等价受控入口调用。
- v1 只允许读取已编译上下文；不得直接调用 Provider、写文件、改批准状态或推进工作流。
- AgentToolExecutor 只返回结构化结果；Harness 持久化报告时获取短租约并重核 `inputHash/worldStateVersion`。推理本身不长期占用项目租约，也不走当前“成功即保存产品 Artifact”的路径。
- `reportArtifactDraft` 是审计工件，不等于产品节点完成，也不等于 QualityDecision。
- `adapterKind: agent` 是推荐接口；若实现采用其他名称，也必须保持与 `internal_capability` 的语义隔离。

## 5. 三类报告的权威边界

### 5.1 ValidationReport

```yaml
ValidationReport:
  reportId: string
  reportDigest: string
  authority: deterministic
  domain: ppt | video | delivery
  stage: string
  target:
    artifactId: string
    artifactVersion: integer
    artifactDigest: string
    productionPath: string
  contract: { id: string, version: string }
  overallStatus: passed | failed | inconclusive
  gates:
    - gateId: string
      validatorId: string
      validatorVersion: string
      status: passed | failed | inconclusive
      failureCode: optional string
      evidenceRefs: [string]
      locators: [TargetLocator]
      responsibleStage: string
  createdAt: timestamp
```

Validator 不评分、不调用模型。证据缺失时必须是 `inconclusive`，不能猜测为通过。

### 5.2 CriticReport

```yaml
CriticReport:
  reportId: string
  reportDigest: string
  authority: advisory_semantic
  status: complete | inconclusive
  domain: ppt | video
  stage: string
  target:
    artifactId: string
    artifactVersion: integer
    artifactDigest: string
    productionPath: string
  validationReportRefs:
    - { reportId: string, digest: string }
  effectiveRubric: { id: string, version: string, digest: string }
  targetLocators: [TargetLocator]
  dimensions:
    - dimensionId: string
      score: 95 | 80 | 60 | 30 | not_scorable
      evidenceRefs: [string]
      rationale: string
  findings:
    - findingId: string
      severity: blocker | major | minor
      locator: TargetLocator
      evidenceRefs: [string]
      responsibleStage: string
      minimalFix: string
      invalidatesDownstream: boolean
  recommendation: pass | repair | block
  createdAt: timestamp
```

`recommendation` 只是建议。Critic 不得重新判定 slideCount、PPTX ZIP、ffprobe、文件 hash、参考图是否真实发送或 lineage 是否陈旧。

Critic 不输出权重。Decision Engine 只从 `effectiveRubric` 的不可变版本/digest 读取权重、阈值、证据要求和评分锚点，模型无权临场改变评分政策。

### 5.3 QualityDecision

```yaml
QualityDecision:
  decisionId: string
  decisionDigest: string
  authority: deterministic_policy
  domain: ppt | video
  stage: string
  scope: artifact | unit | unit_set
  target:
    artifactId: string
    artifactVersion: integer
    artifactDigest: string
    productionPath: string
  targetLocators: [TargetLocator]
  policy: { id: string, version: string }
  effectiveRubric: { id: string, version: string, digest: string }
  validationReportRefs: [{ reportId: string, digest: string }]
  criticReportRefs: [{ reportId: string, digest: string }]
  computed:
    calculationStatus: complete | inconclusive
    totalScore: number
    minimumDimensionScore: number
    blockerCount: integer
    majorCount: integer
    minorCount: integer
  outcome: pass | repair | block
  deliveryEligibility: not_eligible | preview_only | unit_candidate | final_candidate
  nextAction: request_human | repair_unit | repair_upstream | regenerate_evidence | ask_teacher | pause
  reasonCodes: [string]
  createdAt: timestamp
```

唯一权威链：

```text
ValidationReport + CriticReport + QualityDecisionPolicy
  -> deterministic QualityDecisionEngine
  -> QualityDecision
```

所有 report/decision digest 使用版本化 canonical JSON 规则对不可变语义载荷计算；不得只 hash ID，也不得在读取后悄悄重算不同内容。`createdAt` 等非语义字段是否纳入由 canonicalization version 固定。

## 6. HumanDecision 合同

```yaml
HumanDecision:
  decisionId: string
  decisionDigest: string
  projectId: string
  actorUserId: string
  actorRole: teacher | reviewer | admin
  action: approve_final | approve_preview | continue_with_known_issues | request_repair | reject
  gateScope: artifact | unit_set | delivery_package
  artifactId: string
  artifactVersion: integer
  artifactDigest: string
  lineageHash: string
  productionPath: string
  inputHash: string
  qualityDecisionId: string
  qualityDecisionDigest: string
  unitIds: [string]
  unitSetDigest: optional string
  acceptedFindingIds: [string]
  rationale: optional string
  createdAt: timestamp
  expiresAt: optional timestamp
  supersedesDecisionId: optional string
```

失效规则：Artifact version、digest、productionPath、inputHash、lineageHash、QualityDecision、目标 unit 或上游 lineage 任一变化，旧决定立即失效。`unit_set` 必须保存对排序后 unitId + version + digest 计算的 `unitSetDigest`，集合增删、换版或换序后旧决定失效。

- `approve_final` 只允许 `final_candidate` 且 blocker=0、major=0。
- blocker 永不可接受。
- major 只允许继续预览或迭代，不得进入最终包。
- minor 可以随最终版本明确接受。
- 每页/每镜头默认不逐个弹确认；`gateScope` 支持代表样张、镜头集合和最终产物，降低确认疲劳。

## 7. ProjectExecutionLease 合同

```yaml
ProjectExecutionLease:
  leaseId: string
  projectId: string
  scope: project_mutation
  ownerRunId: string
  ownerJobId: optional string
  actionInvocationId: string
  actionOfferId: optional string
  humanDecisionId: optional string
  actionKey: string
  idempotencyKey: string
  inputHash: string
  status: active | released | expired
  fencingToken: integer
  acquiredAt: timestamp
  heartbeatAt: timestamp
  expiresAt: timestamp
  releasedAt: optional timestamp
  releaseReason: optional success | failure | cancel | pause | recovery
```

状态机：

```text
free -> active                 acquire
active -> active               renew heartbeat
active -> released             success | failure | cancel | pause
active -> expired              expiresAt passed
expired -> active              recover with new leaseId and fence+1
```

规则：

1. 同一 project 只能有一个 active mutation slot；不同项目可并发。
2. 每个动作都有 `actionInvocationId`；只有 HumanGate 动作才有 actionOfferId/humanDecisionId。一个已披露批次可以共享租约，批次内 unit 继承同一 actionInvocationId；不同教师意图不能借同一批次并发写。
3. HumanGate、ContractPreValidator 通过后才 acquire，等待教师时不占租约。
4. 相同 `(projectId, idempotencyKey, inputHash)` 复用 active/succeeded Job；相同 key、不同 hash 拒绝。
5. 过期恢复必须增加 `fencingToken`；leased action 导致的**每一次**状态写事务都校验 `leaseId + fencingToken`，包括 providerTaskId、poll state、attempt、Artifact、Node、Job、report 和 observation。
6. 已存在 providerTaskId 时恢复 poll/download，禁止重新 submit。
7. Provider submit 已发出但 taskId 尚未可靠持久化时进入 `submission_unknown`；先用 Provider idempotency key 或查询能力对账，无法确认时暂停人工处理，禁止盲目 resubmit。

失败码：

```text
PROJECT_EXECUTION_BUSY
LEASE_IDEMPOTENCY_CONFLICT
LEASE_NOT_FOUND
LEASE_OWNER_MISMATCH
LEASE_EXPIRED
LEASE_FENCED
LEASE_RENEW_FAILED
LEASE_STORAGE_CONFLICT
PROVIDER_SUBMISSION_UNKNOWN
```

这些工程码只写 observation/日志；教师界面使用可理解摘要。

## 8. FinalDeliveryGate 合同

FinalDeliveryGate 是确定性聚合器，不是 Agent、Critic 或第二个 workflow planner。必需角色由版本化 `DeliveryProfile` 决定。

```yaml
FinalDeliveryGateRequest:
  gateRequestId: string
  projectId: string
  phase: pre_package | post_package
  deliveryProfile: { id: string, version: string, digest: string }
  policy: { id: string, version: string, digest: string }
  expectedWorldStateVersion: integer
  sourceRefs:
    - role: lesson | ppt | classroom_visual | intro_video | other
      artifactId: string
      version: integer
      kind: string
      contentHash: string
      artifactDigest: string
      inputHash: string
      lineageHash: string
      productionPath: string
      deliveryEligibility: string
      validationReportRef: { id: string, digest: string }
      criticReportRef: optional { id: string, digest: string }
      qualityDecisionRef: { id: string, digest: string }
      humanDecisionRef: { id: string, digest: string }
      courseAnchorRef: optional CourseAnchorRef
  preGateReportRef: optional { id: string, digest: string }
  expectedGateInputHash: optional string
  packageIdempotencyKey: optional string
  packageArtifactId: optional string
  packageHash: optional string
  manifestHash: optional string

CourseAnchorRef:
  lessonArtifactId: string
  lessonVersion: integer
  anchorId: string
  anchorDigest: string

FinalDeliveryGateReport:
  gateReportId: string
  gateReportDigest: string
  decision: eligible | blocked
  gateInputHash: string
  hardGateResults: [object]
  failureCodes: [string]
  staleRefs: [string]
  missingRoles: [string]
  manifestCandidate: object
  checkedAt: timestamp
```

陈旧算法：

```text
lineageHash = hash(sorted(
  upstreamRole + artifactId + version + contentHash + upstreamLineageHash
))

gateInputHash = hash(
  deliveryProfile/version/digest + policy/version/digest +
  expectedWorldStateVersion + sorted canonical source refs,
  including artifact/input/lineage/report/decision/human/anchor digests
)

packageIdempotencyKey = hash(projectId + gateInputHash)
```

任一上游选择、版本、内容、报告或教师决定变化，旧 GateReport 和旧最终包转为 stale。post-package 请求必须携带 preGateReportRef、expectedGateInputHash 和 packageIdempotencyKey；包 manifest/provenance 同样写入这些值。获取 package lease 后必须同时重核 `worldStateVersion + gateInputHash`，防止 preflight 到打包之间材料变化。相同 hash 且 post-gate 文件仍有效时复用旧包；新 hash 必须生成新候选并使旧包 stale。

硬门：

- DeliveryProfile 必需角色齐全且为当前 selected version。
- 所有 final 角色均为 `final_eligible`；Fast/Short 的 `preview_only` 必须阻断。
- 文件真实、类型、content hash 和基础结构通过。
- 报告、决定、HumanDecision 绑定同一 Artifact version/digest/path/inputHash。
- 当前上游 tuple 与 lineageHash 一致。
- 视频 `courseAnchor + lessonContextRef` 指向当前 lesson；不要求视频读取 PPT 或复用 PPT 资产。
- post-package ZIP 条目、manifest 和 hash 完全一致。

## 14. QualityDecisionPolicy v1

| 目标 | `pass` | `repair` | `block` |
|---|---|---|---|
| PPT final | Validation 全过；总分 >=85；各维 >=70；blocker=0；major=0 | 75-84、任一维 50-69 或 major>0 | Validation failed/inconclusive、总分 <75、任一维 <50 或 blocker>0 |
| Video shot | Validation 全过；总分 >=80；各维 >=65；blocker=0；major=0 | 70-79、任一维 50-64 或 major>0 | Validation failed/inconclusive、总分 <70、任一维 <50 或 blocker>0 |
| Video final | Validation 全过；总分 >=85；各维 >=70；blocker=0；major=0 | 75-84、任一维 50-69 或 major>0 | Validation failed/inconclusive、总分 <75、任一维 <50 或 blocker>0 |

补充规则：

- 规则计算顺序固定为 `block > repair > pass`。先检查报告完整性/硬门/blocker/阻断阈值，再检查 major/返修阈值，最后才允许 pass；同一输入不能因条件匹配顺序不同得到不同结果。
- 对 PPT final、Video shot、Video final，CriticReport 是必需报告。缺失、digest 不匹配、`inconclusive`、任一必需维度 `not_scorable` 时一律 `block + regenerate_evidence`，禁止用部分分数通过。
- `block` 阻断当前版本，不代表永久终止；必须给 nextAction。
- Fast/Short 无论得分多高都只能 `preview_only`。
- 单镜头通过最多 `unit_candidate`。
- Quality/Full 通过为 `final_candidate`，还需 HumanDecision 才能 `final_eligible`。
- `EffectiveRubric v1` 的基础权重由 Registry 固定并带 digest，Profile v1 不修改权重。Profile 可在生成前增加子标准/证据要求、提高阈值或增加严重度规则；不得删除基础维度、降低底线、降级 blocker 或提高路径上限。权重调优等真实教师样本校准后另发 v2。

## 19. Guard、执行与 Replan 协议

```text
observe current WorldState
plan with Main Agent
resolve ActionOffer control semantics

if tool unknown: block(UNKNOWN_TOOL)
if ContractPreValidator fails: observe -> repair_upstream | ask_teacher
if provider unavailable: observe -> replan path/profile
if IdempotencyLedger has exact active/succeeded (key + inputHash): reuse status/result
if same idempotencyKey has different inputHash: block(IDEMPOTENCY_CONFLICT)
if side effect requires HumanGate and scoped ActionOffer not confirmed: await confirmation
if run/action budget exhausted: pause(CIRCUIT_BREAKER)

if action changes project state:
  acquire ProjectExecutionLease(idempotencyKey, inputHash)
  recheck worldStateVersion + inputHash

act through ToolRouter
stage immutable raw output/blob without advancing product WorldState
run ContractPostValidator -> ValidationReport
fenced promote to needs_review OR quarantine/reject
persist ValidationReport + observation; every write checks leaseId + fencingToken
release/pause project mutation lease

when semantic review required:
  call Delivery Critic on immutable snapshot without long lease -> CriticReport
  run deterministic QualityDecisionEngine -> QualityDecision
  acquire short_persist lease
  recheck worldStateVersion + inputHash + target/report digests
  persist CriticReport + QualityDecision + observation with fencing
  release short lease

return observation to Main Agent -> replan
```

最终包单独执行：

```text
compile FinalDeliveryGateRequest from backend WorldState

exact idempotency lookup by projectId + gateInputHash
HumanGate
acquire lease
recheck worldStateVersion + gateInputHash + preGateReport digest
package
stage package candidate with preGateReportRef + expectedGateInputHash + packageIdempotencyKey
post-package gate
fenced promote/publish or quarantine without advancing final node
release lease
```

## 20. 防过度约束检查

本合同只硬约束：

- 教材事实、儿童安全、隐私和授权。
- 真实文件、版本、hash、血缘、参考资产真实传输。
- pageId/shotId、逐页/逐镜头完整性、交付路径身份。
- 质量证据、报告分权、最终交付资格和项目并发一致性。

本合同不硬编码：

- PPT 必须某种视觉风格、固定样张数或固定模板。
- 视频必须固定镜头数、固定时长、固定模型或固定运镜。
- 每个页面永远只有一个知识点。
- 所有失败都整套重做。
- 教师必须按固定顺序先做 PPT 再做视频。

模型保留的能力包括：选择生产路径、调整软方法、决定样张覆盖、设计创意、选择镜头数量、根据 observation 改道、局部联动返修和向教师提出方向性问题。系统只阻止它伪造事实、越权、重复付费、并发写乱状态或把预览冒充最终交付。

## 21. 实施前验收问题

1. 是否能从同一输入稳定计算相同 QualityDecision？
2. Critic 是否完全不能修改 Validator 结果？
3. 每个问题是否能定位到 page/asset/shot/track/timeline？
4. Fast/Short 是否被机器限制在 `preview_only`？
5. HumanDecision 是否在版本、digest、inputHash 或上游变化后失效？
6. 同项目两个并发写是否只有一个拿到 lease，旧 worker 是否被 fence？
7. FinalDeliveryGate 是否能拦截陈旧血缘、错报告、错人工决定和 ZIP/manifest 不一致？
8. Resolver 是否只处理控制语义，而不再成为第二 capability planner？
9. 缺硬前置或质量失败后，Main Agent 是否能得到 observation 并执行最小 Replan？
10. Profile 是否可以表达风格差异，但不能降低事实、安全和真实交付底线？
