# 交付效果问题诊断

## 1. 总判断

当前产品的架构成熟度高于交付效果成熟度。真实性与安全门已经较强，但“好不好用、好不好看、能不能上课”缺少可计算的定义。

## 2. PPT 问题

### 2.1 叙事入口太晚

现有流程从需求直接进入 PPT 大纲，再进入四层设计。缺少“这套课件要让学生经历什么认知变化”的 Deck Narrative，因此容易得到逐页完整但整体平铺的页面集合。

### 2.2 四层字段只能防缺项，不能保证好设计

当前验证只检查每页是否有“底图、元素、文字、排版”，无法判断：

- 页面是否只有一个教学动作。
- 主视觉是否真的承担解释任务。
- 信息是否在投影距离可读。
- 页面轮廓是否连续重复。
- 页面之间是否形成提问、探索、验证、迁移和总结的节奏。
- 文字是否只是把教案搬上屏幕。

### 2.3 视觉素材与页面任务脱节

课堂图片工具当前只读取 `ppt_draft` 并生成一张通用图片，无法支持“每页先定一个主视觉”的 image-led 工作方式，也没有素材与页面 ID 的强绑定。

### 2.4 PPTX 验真不验效果

当前 PPTX 门禁可以证明文件是真实 ZIP、存在 `presentation.xml`、页数匹配，但不能证明：

- 没有溢出、遮挡、极小字体和不可读配色。
- 图片没有错误裁切或低分辨率放大。
- 每页都落实了设计稿，而非 Provider 自行改写。
- 数学数量、公式、符号和教材结论准确。
- 可编辑性、阅读顺序和无障碍信息完整。

## 3. 视频问题

### 3.1 创意生成与选择之间没有硬门

系统能生成创意主题，但没有“候选集 -> 评分 -> 教师选择 -> 锁定主方案”的结构化 Contract。下游容易把一个未经选择的方案直接展开。

### 3.2 分镜是字段清单，不是导演意图

现有分镜字段包含时长、场景、动作和镜头运动，但缺少每个 beat 的体验目标、情绪、构图世界、视觉焦点、进出场关系和失败判据。

### 3.3 资产连续性不足

当前资产说明最终只生成一张资产图。角色、场景、道具、关键帧和色彩无法形成多视图 Bible，多个片段之间容易漂移。

### 3.4 Provider 输入被压缩得过薄

视频 Provider 提示主要读取片段计划前 1600 个字符。即便上游已经形成分镜和资产图，最终调用也可能没有把角色锁定、关键帧、禁止项、镜头起止状态完整传入。

### 3.5 成片拼接实现不符合媒体语义

当前 `Buffer.concat()` 只把多个 MP4 文件字节相加。MP4 是带索引和时间戳的容器，正确方式应使用 FFmpeg concat demuxer/filter 或统一解码后重编码，并验证总时长、流参数和时间戳连续性。

### 3.6 质量门只看容器

目前视频门只验证 `ftyp` 和 `moov`，无法发现：黑帧、长静帧、画面跳变、对象数量错误、字幕遮挡、旁白不同步、音量过低、角色漂移和课程锚点丢失。

## 4. 样例交付暴露的问题

`2026-07-10-五以内数的认识-完整交付` 的 README 声称包含 PPTX 和验收报告，但实际目录中两者均不存在。视频文件技术规格为 60.032 秒、1920×1080、H.264/AAC，容器合格；然而 FFmpeg 检测到多段 2～5 秒静止，联系表呈现明显的五列卡片式构图，与其自身 `design.md` 中“不是网页/PPT 卡片网格”的要求冲突。

这说明当前链路存在三类断裂：

1. 设计规则写了但没有机器可执行的审查。
2. 技术校验通过但视觉体验没有通过。
3. README/manifest 声称与真实目录没有反向核对。

## 5. 应优先优化的七个杠杆

| 优先级 | 杠杆 | 原因 |
|---:|---|---|
| P0 | 输出 Contract 和质量 Rubric | 决定模型到底优化什么 |
| P0 | 渲染后审查与定向返工 | 生成前无法发现真实版式和媒体问题 |
| P0 | 最终清单反向核验 | 防止“文档声称有、目录实际没有” |
| P1 | PPT Narrative / Video Beat Direction | 解决“完整但平”的根因 |
| P1 | 资产与页面/镜头强绑定 | 降低视觉漂移和素材错用 |
| P1 | 生成者与审查者分离 | 避免模型自检宽松 |
| P2 | Provider 调优和替换 | 应在业务契约稳定后做，否则只是换模型碰运气 |
