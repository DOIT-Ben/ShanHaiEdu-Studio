# v0.3 Zen Workbench 降密度规划

## 1. 第一性原理

当前界面已经干净，但仍然“碎”：左侧项目常驻三行信息，中央多张卡片重复包裹，右侧节点图标与文字同时常驻。v0.3 的目标是把默认可见信息降到一层，把细节放进 hover、详情抽屉和当前节点里。

成功标准：

- 教师第一眼只看到当前项目、当前步骤、当前行动和输入框。
- 左侧项目默认一行主标题加一行轻元信息，不常驻状态徽标。
- 中央不再每段都像卡片，只有当前行动和当前产物保留边界。
- 右侧产物栏是 timeline，不是小按钮列表。
- 三栏使用同一灰阶、同一 radius、同一 transition。
- 交互仍完整：复制、作为输入、详情、确认、失败恢复都保留。

## 2. 复用与参考

复用现有 Next.js、Tailwind、Radix、lucide 和已拆分组件，不增加 UI 库。参考 Codex 类工作台的原则：

- 默认信息少，hover 才露出更多。
- 选中态是浅灰底，不用彩色强调。
- 主要视觉重心是 composer 和当前内容。
- 侧栏是导航，不是报告列表。

## 3. 适配方式

- 全局增加统一 motion token：120-180ms、ease-out、active 轻按压。
- `ProjectSidebar`：行列表进一步降密度，当前项显示二级信息，非当前项只显示课题与更新时间。
- `ChatTranscript`：消息不再全部卡片化，使用窄文本块和轻边框。
- `GenerationPanel`：合并已完成任务和参考依据为“上游依据”轻量段落，当前行动保留为唯一状态块，PPT 产物块降噪。
- `ArtifactRail`：改为 timeline 节点，默认只突出当前节点，其他节点退为灰色短名。
- `PromptComposer`：减轻阴影，按钮改成更低存在感。

## 4. 风险和验证

风险：

- 降低信息密度可能影响状态感知，因此当前项必须清楚。
- 右侧节点过轻可能难以点击，因此保留足够触达面积与 hover preview。
- 中央去卡片化后需要靠间距维持层级。

验证：

- `npm run build` 通过。
- 桌面截图中左侧、中央、右侧不再像三套视觉系统。
- 390px 无横向溢出。
- 点击右侧 PPT 节点仍能打开详情抽屉。
