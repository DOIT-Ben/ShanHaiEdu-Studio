# 互动课件规格与课堂活动模块需求

日期：2026-07-14
状态：已接受的后端基础需求；不改变 V1-9R 的发布阻塞优先级。

## 目标

在现有教师 `Project` 中沉淀可编辑、可验证、可版本化的互动课件规格，作为备课智能体、教师编辑器和未来课堂运行端的共同输入。

## 首版范围

- 课件含元数据、页面和互动活动。
- 活动首批支持单选、多选、判断、填空、拖拽配对五类。
- 每项活动必须有稳定 ID、教学目标映射、题干、答案判定、时长与明确结束条件。
- 规格作为 `interactive_courseware_spec` 版本化 Artifact 保存，沿用现有项目授权与 Artifact 审批。
- 校验失败返回机器可读 locator；教师界面仅映射为可理解的修复提示。

## 不做事项

- 不建立独立课堂、学生、班级、账户、文件或 Provider 系统。
- 不接真实学生作答、实时通信、积分榜、白板或发布流程。
- 不把确定性模板或 mock 数据标记为智能生成或正式课件。
- 不让模型绕过 Node Contract、ToolRouter、PlanGuard、HumanGate 或 Quality Gate。

## 智能体接入边界

后续 `interactive_courseware_spec_generate` 只能由 Studio Main Agent 经 Capability Registry 和 ToolRouter 调用；模型输出只是候选规格，必须通过本规格校验并以草稿 Artifact 持久化。教师批准后，才允许课堂运行模块读取。

## 成功标准

1. 有效规格能稳定校验并保存为项目内新版本。
2. 五类活动的答案都可由确定性规则判定。
3. 规格不含 Provider、密钥、学生个人信息或课堂运行事实。
4. 后续播放器可只依赖批准规格渲染，不读取对话文本猜测题目。
