# 质量评估与自动返工机制

## 1. 两类真值

### Artifact Truth

回答“文件是不是真的”：存在、可读、格式正确、页数/时长/尺寸匹配、哈希可追溯。

### Delivery Effect Truth

回答“文件能不能用于课堂”：内容准确、教学动作成立、视觉可读、叙事连贯、媒体连续、教师可操作。

两类真值必须同时通过，不能互相替代。

## 2. 评分维度

所有维度按 0～4 分评分：

- 0：缺失或严重错误。
- 1：存在但不可用，需要重做。
- 2：勉强可用，需要明显修改。
- 3：达到课堂交付标准。
- 4：表现优秀，可作为复用范例。

### PPT Rubric

| 维度 | 权重 | 硬门 |
|---|---:|---|
| 教材与数学准确性 | 20 | 不得低于 3 |
| 学习叙事与页面推进 | 15 | 不得低于 2 |
| 单页教学动作清晰度 | 15 | 不得低于 3 |
| 主视觉解释力 | 15 | 不得低于 2 |
| 投影可读性与无障碍 | 15 | 不得低于 3 |
| 版式节奏与一致性 | 10 | 不得低于 2 |
| 可编辑性与素材真实性 | 10 | 不得低于 3 |

总分至少 80/100，且所有硬门通过。

### 视频 Rubric

| 维度 | 权重 | 硬门 |
|---|---:|---|
| 内容/数量/课堂边界准确性 | 20 | 不得低于 3 |
| 独立创意与开场钩子 | 15 | 不得低于 2 |
| 叙事节拍与镜头动势 | 15 | 不得低于 2 |
| 角色/场景/资产连续性 | 15 | 不得低于 3 |
| 画面可读性与字幕 | 10 | 不得低于 3 |
| 旁白、音乐和声音 | 10 | 不得低于 3 |
| 技术质量与真实合成 | 10 | 不得低于 3 |
| 课程锚点与课堂落点 | 5 | 不得低于 3 |

总分至少 80/100，且所有硬门通过。

## 3. 审查证据要求

Critic 的每条失败必须包含：

- 可定位目标：page_id、shot_id 或时间段。
- 观察证据：渲染图、像素位置、字幕时间、ffprobe 输出或内容片段。
- 期望与实际差异。
- 影响：学生理解、教师操作、视觉信任或文件可用性。
- 最小修复指令。
- 需要重跑的节点，禁止默认整链重跑。

禁止使用“整体还可以”“建议更美观”“加强趣味性”等不可执行评语。

## 4. 自动检查层

### PPT

- 解压并检查 Office Open XML。
- 真实 slideCount 与 PageSpec 数量一致。
- LibreOffice 转 PDF、逐页 PNG。
- 元素边界、文字溢出、空白页、重复页检测。
- 字体大小、对比度、图片有效像素和长宽比。
- OCR/文本抽取与 PageSpec visible text 对照。
- 页面感知哈希检测连续重复轮廓。

### 视频

- `ffprobe` 检查流、时长、帧率、分辨率、编码和音频。
- 完整解码检查。
- `blackdetect`、`freezedetect`、场景变化和帧差分析。
- 音量、峰值、静音段和旁白/BGM 比例。
- 字幕时间轴重叠、越界、阅读速度和末尾残留。
- 按 ShotSpec 时间点抽帧与联系表。
- 成片时长与片段时长总和及转场预算对照。

## 5. 多模态审查层

采用单独 Critic 模型，输入只包含：

- 被审查产物或渲染证据。
- 对应 Contract 和 Rubric。
- 必要教材证据。
- 自动检查结果。

不提供生成者的解释和自评分，避免被其理由锚定。

## 6. 返工路由

```text
Finding
  -> classify(target, dimension, severity)
  -> choose smallest repair node
  -> preserve passed dependencies
  -> regenerate / relayout / remix / reassemble
  -> rerun affected checks
  -> rerun cross-delivery checks only when shared contract changed
```

### 示例

| 发现 | 最小返工节点 |
|---|---|
| PPT 第 5 页字太小 | page layout repair |
| 第 7 页主视觉与概念无关 | page asset regenerate |
| 第 8 页数量错误 | local math layer repair |
| 视频 Shot 03 角色变脸 | shot regenerate with continuity refs |
| 字幕遮挡主体 | caption layout repair |
| BGM 盖住旁白 | audio remix |
| 成片只播放第一段 | timeline reassemble |
| README 声称文件不存在 | manifest regenerate，阻止交付 |

## 7. 最终审计

最终 Auditor 从真实目录重新建立清单：

1. 枚举要求的所有交付物。
2. 检查文件存在、大小、格式和哈希。
3. 读取每个质量门结果及版本。
4. 对照 README/manifest，发现差异即失败。
5. 输出已通过、缺失、失败和不可验证四类状态。

不允许 README、数据库状态或模型文字单独证明交付完成。
