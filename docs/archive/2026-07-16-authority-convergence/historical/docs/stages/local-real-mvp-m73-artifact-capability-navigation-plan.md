# Local Real MVP M73 产物能力导航与安静抽屉计划

更新时间：2026-07-11

## 1. 目标

将按产物数量增长的桌面圆形节点 rail 收敛为固定能力分组入口，并统一桌面分组抽屉、移动抽屉和成果阅读区的低噪声体验，同时保持所有真实产物回调和下载能力。

## 2. 范围与边界

- 只修改 RQ-016、M73 plan/test-plan、产物导航/展示组件、`MediaWorkbench` 和直接相关测试。
- 能力组固定为教案与教材、PPT、图片、视频、最终交付；另设全部产物入口，总数最多 6。
- 复用现有 `lucide-react`、Sheet、Tooltip、ScrollArea、Button 和状态元数据，不增加依赖。
- 不修改后端、产物数据契约、对话、认证、反馈或生成逻辑。

## 3. 设计与实施

1. 提取纯能力分组与状态聚合模块，使用 `ArtifactKind` 显式映射；未知 kind 安全归入教案与教材，避免产物不可达。
2. 桌面 rail 仅渲染有产物的能力组和“全部产物”；入口固定 44px，显示图标、数量、aria-label、tooltip 和最需关注状态。
3. 单产物能力组直接打开侧边阅读；多产物组和“全部”打开同一 drawer，并把初始能力筛选传入抽屉。
4. drawer 使用“备课成果”头部、总数/需处理数、紧凑 chips、安静列表行、完整空态与筛选无结果态。
5. rail 不再按产物绘制节点或连接线；打开侧边阅读/抽屉时关闭 hover 预览。
6. 预览卡只保留一个“打开阅读”主动作；复制、作为输入和下载继续保留在侧边阅读/完整详情，确保真实 wiring 不丢失。
7. 轻调侧边阅读和详情背景、边框与底部操作布局；删除仅在 ArtifactRail 中已确认未使用的 `onRegenerate` prop。
8. 在 `MediaWorkbench` 记录详情是否来自成果抽屉；来自抽屉时显示“返回备课成果”，返回后恢复原能力筛选，右上角关闭仍直接退出成果浏览。

## 4. 风险与回退

- 风险：kind 漏映射导致产物入口消失。控制：映射覆盖联合类型，并以纯函数测试所有 kind；全部入口始终可达。
- 风险：状态聚合误导。控制：固定优先级并测试 blocked、review/stale、in-progress 和稳定态组合。
- 风险：既有 M72 工作区改动被覆盖。控制：仅局部编辑相关文件，保留现有 diff，不做格式化或回滚。
- 风险：把所有详情都误判为来自抽屉。控制：只在 drawer 列表回调中设置来源，侧边阅读进入详情时主动清除来源。
- 回退：可独立回退 M73 文档、分组模块、产物组件和 `MediaWorkbench` 的分组状态，不涉及数据迁移。

## 5. 成功标准

- rail 最多 6 个固定尺寸入口且不随项目产物数量溢出。
- 单项直开、多项/全部进 drawer，预览不与阅读层叠加。
- drawer 详情可返回原能力分组，返回和关闭语义分离，无需关闭重开。
- 桌面和窄屏抽屉均为能力筛选与安静列表，不出现“线性产物”。
- 定向测试、`tsc --noEmit`、`npm test`、`npm run build`、`git diff --check` 通过；浏览器完成桌面/390px 验收，或明确记录登录阻塞。
