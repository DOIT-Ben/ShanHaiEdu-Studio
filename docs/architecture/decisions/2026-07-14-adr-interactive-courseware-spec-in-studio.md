# ADR：互动课件规格在 Studio 内部孵化

日期：2026-07-14
状态：accepted

## 决策

互动课件后端以 Studio 的 `Project -> Artifact` 链路实现，首个事实对象为 `interactive_courseware_spec`。不从静态前端包、生态治理仓或 OpenCode Runtime 新建独立服务。

## 理由

Studio 已拥有教师项目、授权、版本化 Artifact、对话、质量门、工具路由与审计。互动课件需要复用同一项目上下文和审批事实；复制这些能力会形成第二产品主线。

## 取舍

首版只实现严格规格与确定性校验，不实现实时课堂状态。这样先保证课件内容可编辑、可审查、可复用，后续课堂会话可独立增加且不污染课件版本。

## 风险与回退

风险：规格过早扩张为课堂运行引擎。缓解：首版禁止学生数据、实时通信和 Provider 调用。回退：停止注册后续 Capability，不删除已有 Artifact；已保存规格继续作为普通草稿保留。

## 验证

通过合同测试验证五类活动、ID 唯一性、答案可判定性、教学目标映射和结束条件；通过项目内 Artifact 保存测试验证权限与版本化。
